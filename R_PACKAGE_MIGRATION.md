# ChemReg → R Package Migration Analysis

**Date:** 2026-04-13
**Goal:** Enable the project to function as both an interactive Shiny app and a headless R package, so cleaning and curation functions can be called directly from scripts without launching the UI.

---

## Current State

The project is already well-structured for this transition. Core logic lives in `R/*.R` (pure functions), and Shiny-specific code is isolated to `app.R` and `R/modules/`. The main gaps are packaging conventions rather than architectural problems.

```
R/
  cleaning_pipeline.R   # Pure cleaning functions — the main payload
  cleaning_reference.R  # Reference list loading
  config_import.R       # Import configuration helpers
  consensus.R           # DTXSID consensus logic
  curation.R            # CompTox API + deduplication pipeline
  data_detection.R      # Frontmatter detection ensemble
  export_helpers.R      # Output formatting
  file_handlers.R       # File reading and validation
  modules/              # Shiny UI/server modules (8 files)
app.R                   # Shiny entry point (sources all R/ files)
scripts/
  curate_dataset.R      # Standalone prototype — shows the headless path
```

### 2. Remove `library()` calls from `R/*.R`

`library()` inside package source files pollutes the user's search path and will cause `R CMD check` warnings. The following files have bare `library()` calls:

| File | Libraries called |
|------|-----------------|
| `R/cleaning_pipeline.R` | `dplyr`, `stringr`, `stringi`, `tibble` |
| `R/cleaning_reference.R` | `fs`, `tibble`, `dplyr` |
| `R/consensus.R` | `dplyr`, `tibble` |

**Fix:** Remove all `library()` calls from these files. Since the codebase already uses `pkg::function()` notation extensively, the simplest approach is to verify all calls are namespaced and drop the `library()` lines. Add the packages to `Imports:` in `DESCRIPTION` instead.

For any calls that are *not* yet namespaced, either add `::` prefixes or use `@importFrom` roxygen tags.

### 3. Move the Shiny app to `inst/app/`

R packages conventionally place runnable apps under `inst/app/`. The app is then launched via a wrapper function that uses `system.file()` to locate it at runtime.

**New directory structure:**
```
inst/
  app/
    app.R          # Moved from project root
  extdata/
    reference_cache/   # Moved from data/reference_cache/
```

**New launcher function** — add as `R/run_app.R`:
```r
#' Launch the ChemReg Shiny application
#'
#' @param ... Arguments passed to [shiny::runApp()]
#' @export
run_app <- function(...) {
  app_dir <- system.file("app", package = "chemreg")
  if (app_dir == "") stop("Could not find app directory. Is the chemreg package installed?")
  shiny::runApp(app_dir, ...)
}
```

**Update `inst/app/app.R`:** The current `app.R` sources all `R/` files with a `for` loop. Inside a package, those functions are already loaded — remove the `source()` loop entirely. The `library()` block at the top can also shrink to just the Shiny stack (the rest comes from the package namespace).

### 4. Fix `here::here()` references to package data

`app.R` line 40 loads reference lists from a path constructed with `here::here()`:

```r
reference_lists <- load_all_reference_lists(here::here("data", "reference_cache"))
```

`here::here()` resolves relative to the project root during development but breaks when the package is installed elsewhere. Replace with:

```r
reference_lists <- load_all_reference_lists(
  system.file("extdata", "reference_cache", package = "chemreg")
)
```

This requires moving `data/reference_cache/` → `inst/extdata/reference_cache/`. Any other `here::here("data", ...)` calls in `R/*.R` need the same treatment.

### 5. Add a headless entry point

The primary motivation. `scripts/curate_dataset.R` already demonstrates the pipeline — this formalizes it as an exported function:

```r
#' Run the curation pipeline without the Shiny UI
#'
#' Reads a file, detects frontmatter, cleans chemical names and CAS numbers,
#' and writes a curated output file.
#'
#' @param input_path Path to input CSV or XLSX file
#' @param output_path Path to write the cleaned XLSX output
#' @param tag_map Named list mapping column names to roles: "Name", "CASRN", or "Other".
#'   If NULL, the function attempts to infer column roles from headers.
#' @param skip_flags Character vector of cleaning flags that should bypass API lookup.
#'   Defaults to `c("isotope_match", "chiral_match")`.
#' @return A list with `data` (cleaned data frame) and `audit_trail` (change log), invisibly.
#' @export
curate_headless <- function(input_path, output_path, tag_map = NULL, skip_flags = c("isotope_match", "chiral_match")) {
  raw        <- safely_read_file(input_path)
  detected   <- detect_data_start(raw$data)
  clean      <- extract_clean_data(raw$data, detected)
  cleaned    <- run_cleaning_pipeline(clean)
  deduped    <- deduplicate_tagged_columns(cleaned$data, tag_map, skip_flags = skip_flags)
  # ... wire through CompTox search and consensus classification
  writexl::write_xlsx(result$data, output_path)
  invisible(result)
}
```

The exact wiring follows `scripts/curate_dataset.R`. The key value is that all the intermediate steps (detection, cleaning, dedup, API, consensus) become callable individually too, since they're already exported functions.

### 6. Migrate tests to `tests/testthat/`

Current test files live in `tests/test_*.R`. The `testthat` package expects:

```
tests/
  testthat.R           # Test runner (one-liner)
  testthat/
    test-cleaning.R    # Renamed from test_*.R (dashes, not underscores)
    test-detection.R
    test-enrichment.R
    ...
```

`tests/testthat.R` content:
```r
library(testthat)
library(chemreg)
test_check("chemreg")
```

Test files themselves need no changes beyond the rename — `testthat` syntax is identical.

## What Does NOT Need to Change

- All function implementations in `R/*.R` — logic is unchanged
- `R/modules/` Shiny modules — they stay in place
- `air.toml`, `.gitignore`, CI config
- `load_packages.R` — keep for dev convenience, just never `source()` it from package code
