## MEDIUM -- Should fix

1. **Multi-locant tests don't assert `synonym_count`** (`tests/test_name_cleaning.R:325-367`) -- Tests verify row count but don't explicitly check `synonym_count == 1L` for protected names, which would catch a regression where names survive unsplit but the count metadata is wrong.

2. **Roman numeral XII cap undocumented in tests** (`R/cleaning_pipeline.R:17`) -- `ROMAN_NUMERAL_PATTERN` matches I–XII only. The boundary isn't tested. Add a negative test confirming XIII is *not* protected, so future maintainers know the limit is intentional.

3. **Missing edge case: `"chromium(III)"` without space before paren** (`tests/test_name_cleaning.R:641-688`) -- All roman numeral tests use `"chromium (III)"` (with space). Real databases sometimes omit the space. Should verify behavior and add a test.

## LOW -- Nice to have

4. **List accumulation pattern in `split_synonyms` and `strip_reference_terms`** (`R/cleaning_pipeline.R:847,777`) -- Uses `expanded_rows[[length(expanded_rows) + 1]]` append pattern. O(n) per append. Negligible for typical chemical inventories.

5. **No test for combined letter-comma-letter + digit-comma-digit** (`tests/test_name_cleaning.R`) -- Names like `"N,N-2,4-dimethylpyrrole"` exercise both protection branches simultaneously. Worth adding as an edge case.

6. **Repeat-until-stable loop uses magic number** (`R/cleaning_pipeline.R:859`) -- `seq_len(10)` is safe and correct but a named constant like `MAX_PROTECTION_ITERS <- 10L` would document intent.

7. **`VI{0,3}` in ROMAN_NUMERAL_PATTERN redundantly covers `V`** (`R/cleaning_pipeline.R:17`) -- The alternation includes both `V` and `VI{0,3}` (which expands to V, VI, VII, VIII). Standalone `V` is redundant. Not a bug — regex engine handles it — but could confuse maintainers.
