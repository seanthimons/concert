Use DT's `escape = FALSE` option and inject HTML select elements into the Resolution column. For each row:
- If consensus_status == "disagree" AND .pinned == FALSE: render a `<select>` with options from `get_resolution_options()` plus a blank default.

- Otherwise: empty cell or dash.

The select element should have a class like `resolve-select` and a `data-row` attribute with the row index.

Add a JS callback (via `tags$script` in the UI or DT `initComplete` callback) that listens for change events on `.resolve-select` elements and calls `Shiny.setInputValue("resolve_row_choice", {row: rowIdx, column: selectedValue}, {priority: "event"})`.

Add `observeEvent(input$resolve_row_choice, { ... })` in server that:
- Calls `resolve_row(data_store$resolution_state, row_idx, chosen_column, data_store$dtxsid_cols)`
- Updates `data_store$resolution_state` with the result
- Table re-renders automatically since it reads from resolution_state

`priority_controls` renderUI: For each column in `data_store$priority_order`, show a row with:
- Column name (stripped of dtxsid_ prefix for readability)
- Up button (actionButton with id `priority_up_{i}`)
- Down button (actionButton with id `priority_down_{i}`)

Use `lapply` over `seq_along(data_store$priority_order)` to generate the UI.

Handle up/down buttons: Use `observeEvent` with a pattern. Since dynamic IDs are tricky, use a simpler approach: render the priority list as a set of `actionLink` elements and handle via JS, OR use `shiny::req` with explicit up/down IDs.

Simpler alternative: Use a `selectInput` with `multiple = TRUE, selectize = TRUE` and the sortable option. Or just use numbered text inputs showing the order, with up/down actionButtons per slot.

Practical approach: Since there are typically only 2-4 tagged columns, render a simple ordered list with up/down buttons per item. Store priority_order as a reactiveVal or in data_store. Each up/down button swaps adjacent items.

For the button observers, use a single observer per direction that reads which button was clicked via `input$last_priority_up` / `input$last_priority_down` (set via JS on click). Or generate observers dynamically with `lapply`.

`observeEvent(input$apply_priority, { ... })`:
