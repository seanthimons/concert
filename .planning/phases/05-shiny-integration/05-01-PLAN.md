Delete all existing content in R/curation.R. Replace with a single orchestrator function:

```r
run_curation_pipeline <- function(clean_data, column_tags, progress_callback = NULL)
```

Parameters:
- `clean_data`: the cleaned data frame (data_store$clean)
- `column_tags`: named list (col_name -> "Name"|"CASRN"|"Other")
- `progress_callback`: optional function(stage, message) for reporting progress to Shiny

Implementation steps inside the function:

1. **Dedup stage**: Call `deduplicate_tagged_columns(clean_data, column_tags)`. Fire progress_callback("dedup", paste0("Deduplicated: ", length(dedup$unique_names), " unique names, ", length(dedup$unique_cas), " unique CAS")). Return early with just dedup counts if a `dedup_only = TRUE` parameter is passed (for the preview feature).

2. **Tiered search stage**: Call `run_tiered_search(dedup_result)`. Fire progress callbacks at each sub-stage. To provide per-tier progress, replace the single `run_tiered_search` call with inline orchestration:
   - Call `search_exact(dedup$unique_names)`. Callback: "Exact match: X/Y found, Z falling back..."
   - Compute missed names. Call `search_starts_with(missed_names)`. Callback: "Starts-with: X more found..."
   - Call `validate_and_lookup_cas(dedup$unique_cas)`. Callback: "CAS validated: X valid, Y invalid..."
   - Combine results into the same tibble format as `run_tiered_search` produces (searchValue, dtxsid, preferredName, searchName, rank, source_tier).

3. **Map results**: Call `map_results_to_rows(clean_data, dedup$dedup_key_map, combined_results)`.

4. **Consensus stage**: Call `find_dtxsid_cols(mapped_df)` then `classify_consensus(mapped_df, dtxsid_cols)`. Callback: "Consensus: X agree, Y disagree, Z partial..."

5. **Init resolution**: Call `init_resolution_state(classified_df)`.

6. **Return** a list:
   - `results`: the fully classified+resolved data frame
   - `dedup_summary`: list(n_names = ..., n_cas = ...)
   - `search_summary`: list(n_exact = ..., n_starts_with = ..., n_cas_valid = ..., n_miss = ...)
   - `consensus_summary`: list(n_agree = ..., n_disagree = ..., n_agree_caveat = ..., n_single = ..., n_error = ...)

Also add a helper function:

```r
get_dedup_preview <- function(clean_data, column_tags)
```

That calls `deduplicate_tagged_columns` and returns just the unique counts (for the preview feature before the user clicks Start Curation).

Do NOT use `library()` calls at the top of the file -- the functions from prototype_pipeline.R and consensus.R are already sourced by app.R. Add a comment header noting the file depends on prototype_pipeline.R and consensus.R being sourced first.
