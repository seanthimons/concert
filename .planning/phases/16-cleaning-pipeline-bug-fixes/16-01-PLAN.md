detect_bare_formulas(df, name_cols)
  - Uses ComptoxR:::create_formula_extractor_final() to get validator_regex
  - Strips spaces and dots from name, then tests ^validator_regex$
  - BUG: "Naphthalene" -> "Naphthalene" matches because Na+ph+tha+le+ne looks like element symbols
  - FIX APPROACH: Add a pre-check that rejects names containing lowercase-only sequences of 2+ chars
    that aren't element symbols. More precisely: if the original name (before space/dot removal) contains
    any lowercase letter NOT immediately following an uppercase letter (i.e., has words, not just element
    symbols like NaCl), skip formula detection. A simple heuristic: if the name contains any run of 2+
    lowercase letters, it's a word, not a formula. Formulas have patterns like "Na", "Cl", "SO4" -
    single lowercase after uppercase, or digits, never "aph", "tha", "ene", "odium", etc.

flag_reference_matches(df, name_cols, reference_list, flag_type, flag_label)
  - Pass 2 uses str_detect(original_value, regex(ref_term, ignore_case=TRUE))
  - BUG: stop word "na" matches "Naphthalene" via substring
  - FIX APPROACH: Wrap ref_term in word boundaries \b for Pass 2 substring matching.
    Change: str_detect(original_value, regex(paste0("\\b", ref_term, "\\b"), ignore_case=TRUE))
    This makes "na" only match standalone "na", not inside "Naphthalene".
    Note: block_patterns use regex patterns (^proprietary, etc.) - these should NOT get word boundaries
    since they're already regex patterns. The fix should only apply when ref_terms are plain words.
    Since flag_reference_matches is generic, add word boundaries unconditionally in Pass 2 -
    block_patterns already use ^ anchors which won't be affected by \b wrapping.

split_synonyms(df, name_cols, tag_map)
  - Protects digit,digit (@@@ placeholder) and ", digit" (%%% placeholder)
  - BUG: "N,N-Dimethylformamide" splits at the comma because N,N is letter,letter
  - FIX APPROACH: Add a new protection step before split: protect letter-comma-letter patterns
    where both letters are uppercase single chars typical of IUPAC prefixes.
    Pattern: ([A-Za-z]),([A-Za-z]) -> replace comma with @@@ placeholder (reuse existing).
    More specifically: single uppercase letter, comma, single uppercase letter followed by hyphen:
    ([A-Z]),([A-Z]-) to be safe. Actually, the simplest correct fix: protect any pattern where

    Use: str_replace_all(protected_name, "(?<=[A-Za-z]),(?=[A-Za-z](?:[,-]|$))", "@@@")
