  <name>Task 1: Add validate_manual_dtxsids() and merge_retry_results() to R/curation.R</name>
  <files>R/curation.R</files>
  <action>
Add two new exported functions at the end of R/curation.R:

**validate_manual_dtxsids(dtxsids, batch_size = 20, delay_sec = 1)**
- Takes character vector of DTXSID strings (e.g., "DTXSID7020182")
- Deduplicates input DTXSIDs before API call
- Splits into batches of `batch_size` with `delay_sec` between batches (rate limiting)
- For each batch, calls `ComptoxR::ct_chemical_search_equal_bulk()` (same pattern as existing `search_exact()` in this file)
- Returns tibble with columns: searchValue, dtxsid, preferredName, rank, is_valid (logical)
- For DTXSIDs not found in API response, include row with is_valid = FALSE and NA for dtxsid/preferredName
- Wrap API call in tryCatch — on error, mark all as invalid with warning message
- Use `purrr::safely()` pattern consistent with existing codebase error handling

**merge_retry_results(original_state, retry_results, selected_row_indices, tags_changed = FALSE)**
- Takes original resolution_state df, retry pipeline results df, integer vector of original row indices that were re-curated, and boolean indicating if tags were changed
- CRITICAL: Skip any row where `original_state$.pinned[row_idx]` is TRUE (safety check)
- If `tags_changed = TRUE`: add new columns from retry_results that don't exist in original_state (initialize with NA for non-selected rows)
- For each selected row index, update consensus columns: consensus_dtxsid, consensus_status, consensus_source, qc_tier
- Also update per-column lookup columns: dtxsid_*, preferredName_*, rank_*, source_tier_* (match by column name suffix)
- For rows that were error before AND are still error after retry: set consensus_status = "unresolvable"
- Preserve original row order (operate by index, never sort)
- Return the updated original_state data frame
