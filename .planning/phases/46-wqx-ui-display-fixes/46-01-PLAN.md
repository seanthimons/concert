<objective>
Fix three WQX UI display gaps in mod_review_results.R so that WQX-resolved rows display correctly in the Shiny review panel.

Purpose: Phase 45 wired WQX matching into the pipeline and consensus classification, but the review module's helper functions predate WQX and lack branches for the "wqx" status and "wqx_*" source tiers. This causes WQX rows to be invisible in value_boxes, show blank Resolution cells, and display misleading "No Match" labels.

Output: Updated R/mod_review_results.R with WQX support in all three helper functions plus status badge/row styling, and a new test file validating the helpers with WQX inputs.
</objective>
