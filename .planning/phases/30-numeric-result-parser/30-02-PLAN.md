       - Purpose: detect if remainder contains a range hyphen (not a negative sign or exponent)
       - Pre-guard logic (order matters):
         (a) If `qualifier` is not "" → NOT a range (per D-04), return as single value
         (b) Normalize: the remainder has already been through `normalize_numeric_string()`, so Fortran exponents are already `e` notation
         (c) Range regex: `^(-?\d+\.?\d*(?:[eE][+-]?\d+)?)\s*-\s*(-?\d+\.?\d*(?:[eE][+-]?\d+)?)$`

             Key: the hyphen must be between two complete numbers (not after `e`/`E`)
         (d) If regex matches → extract low and high as numeric, compute mid = (low + high) / 2
