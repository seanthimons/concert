<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Shiny server | Modal and batch flag inputs mutate `resolution_state` |
| Reactable selection -> row mutation | Client-side selected display rows are mapped back to source rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-04 | Tampering | batch selected rows | mitigate | Recompute selected rows server-side from reactable state and validate row indices in `set_row_flags()`. |
| T-51-05 | Tampering | arbitrary flag values | mitigate | Modal and batch handlers call Plan 01 validation helpers; invalid values error before state mutation. |
| T-51-06 | Denial of Service | large batch selection | accept | Batch mutation is vectorized over selected visible rows and bounded by table size in the user's session. |
</threat_model>
