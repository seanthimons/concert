<objective>
Extend the Tag Columns module to support numeric and regulatory data columns, implement
tag-type dispatch so each downstream pipeline receives only its relevant tags, and add
independent cascade resets per tag type.

Purpose: Phase 33 bridges the pure-R harmonization engine (Phases 29-32) to the Shiny UI
by enabling users to tag numeric/study columns alongside chemical columns. Without this,
harmonization has no column assignments to operate on.

Output: Modified mod_tag_columns.R with optgroups + dispatch, modified app.R with granular
cascade resets, and unit tests for the dispatch/validation logic.
</objective>
