**Fix 1 (GAP-1): Deduplicate wqx_confidence colDef (around line 769)**

The current code uses `grep("^wqx_confidence", names(df_display), value = TRUE)` and creates a colDef for EVERY matching column. In multi-tag mode (e.g., Name + CASRN), this matches both `wqx_confidence_Chemical` and `wqx_confidence_CASRN`. The CASRN column is all-NA (since WQX only matches Name-tagged columns), producing a useless duplicate "WQX Conf." column.

Replace the current grep + for loop block (lines 766-782) with:

```r
# WQX confidence column (fuzzy similarity score; NA for exact/alias rows)
# In multi-tag mode, grep finds wqx_confidence_Chemical AND wqx_confidence_CASRN.
# Only show columns that have at least one non-NA value (WQX only matches Name-tagged columns).
wqx_conf_cols <- grep("^wqx_confidence", names(df_display), value = TRUE)
wqx_conf_visible <- Filter(function(col) any(!is.na(df_display[[col]])), wqx_conf_cols)
# Hide all-NA wqx_confidence columns (e.g., wqx_confidence_CASRN in multi-tag mode)
wqx_conf_hidden <- setdiff(wqx_conf_cols, wqx_conf_visible)
for (whc in wqx_conf_hidden) {
  col_defs[[whc]] <- reactable::colDef(show = FALSE)
}
for (wcc in wqx_conf_visible) {
  col_defs[[wcc]] <- reactable::colDef(
    name = "WQX Conf.",
    minWidth = 80,
    align = "right",
    cell = function(value, index) {
      if (is.na(value)) {
        return("")
      }
      formatC(value, digits = 2, format = "f")
    }
  )
}
```

Key changes from current code:
- Added `Filter()` line to split columns into visible (has data) vs hidden (all-NA)
- Added explicit `colDef(show = FALSE)` for all-NA columns so they don't render
- The visible colDef loop is unchanged

**Fix 2 (GAP-2): Wrap updateSelectizeInput in session$onFlushed (around line 1880-1886)**

The current code calls `updateSelectizeInput(session, "wqx_typeahead", choices = wqx_choices, server = TRUE)` immediately after `showModal()`. The modal HTML is queued for rendering but the selectize.js widget may not be fully initialized in the DOM when `updateSelectizeInput` fires. The server-side callback registration fails silently.

Replace lines 1880-1886 with:

```r
# Load WQX dictionary and populate selectize
# MUST use onFlushed to ensure modal DOM (and selectize.js widget) is fully initialized
# before reconfiguring for server-side rendering (fixes GAP-2: search box not accepting input)
cache_dir <- system.file("extdata", "reference_cache", package = "chemreg")
wqx_dict <- load_wqx_dictionary(cache_dir)
display_type <- ifelse(wqx_dict$type == "canonical", "canonical", "alias")
wqx_labels <- paste0(wqx_dict$name, " (", display_type, ")")
wqx_choices <- stats::setNames(wqx_dict$canonical_name, wqx_labels)
session$onFlushed(function() {
  updateSelectizeInput(session, "wqx_typeahead", choices = wqx_choices, server = TRUE)
}, once = TRUE)
```

The key change is wrapping the `updateSelectizeInput` call inside `session$onFlushed(once = TRUE)`. This ensures:
1. The `showModal()` render message is flushed to the client first
2. The browser creates the modal DOM and initializes the selectize.js widget
3. Only then does the server send the server-side choice configuration

The dictionary loading stays outside the callback (it runs synchronously and the result is captured in the closure).

test_that("wqx_conf_cols Filter removes all-NA columns in multi-tag mode", {
