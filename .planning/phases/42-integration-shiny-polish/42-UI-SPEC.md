## Spacing Scale

Declared values (must be multiples of 4). All values expressed as Bootstrap 5 utility classes
or inline pixel values in R Shiny tag expressions.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `m-1`, `p-1`) | Icon gaps, badge internal padding, inline spacing |
| sm | 8px (`gap-2`, `m-2`, `p-2`) | Compact element spacing, chip margin |
| md | 16px (`gap-3`, `m-3`, `p-3`) | Default element spacing, modal field gaps |
| lg | 24px (`gap-4`, `py-4`) | Section padding, accordion section gaps |
| xl | 32px (`py-5` approx) | Modal footer area, top-of-tab breathing room |
| 2xl | 48px (`py-5`) | Empty state vertical centering padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Pre-flight checklist rows: 12px vertical padding per row (between sm and md) to keep the
  list scannable without wasting height in a modal
- Touch targets on action buttons: minimum 36px height (Bootstrap btn-sm default is sufficient)

## Color

Palette derived from Bootstrap 5 Flatly bootswatch with primary override `#007bff`.

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `#ffffff` / `#f8f9fa` | Page background, modal body, table rows | Bootstrap Flatly surface |
| Secondary (30%) | `#e9ecef` / `#dee2e6` | Chip container background, table striping, card backgrounds | Existing CSS in mod_clean_data.R line 21 |
| Accent (10%) | `#007bff` (Bootstrap primary) | Reserved list below | inst/app/app.R line 48 |
| Destructive | `#dc3545` (Bootstrap danger) | "Remove" chip button hover (`:hover { color: red }`), no destructive actions in this phase | Existing chip CSS |
| Semantic: success | `#28a745` (Bootstrap success) | "Run Pipeline" button class `btn-success` (inherits from existing "Run Cleaning" / "Run Harmonization" buttons) | mod_clean_data.R line 66, mod_harmonize.R line 78 |
| Semantic: info | `#17a2b8` (Bootstrap info) | User-source badge in media table | D-12 (CONTEXT.md) |
| Semantic: muted | `#6c757d` (Bootstrap secondary/muted) | AMOS-source badge, empty-state text, helper text | Existing pattern |

Accent (`#007bff`) reserved for:
1. "Save Mapping" / "Save" buttons inside modals (`btn-primary`)
2. "Run now" action link in re-run prompt notification
3. "Open full modal" link in zero-changes notification (D-04)
4. Any `btn-primary` confirmation buttons in override dialogs (D-13)
5. Active/focused nav-link highlight (Bootstrap default)

Accent must NOT be used for: table row highlighting, badge backgrounds for AMOS entries,
informational text, or general hyperlinks outside the five reserved elements above.

## Component Inventory

New or modified components this phase, all following existing bslib/shiny patterns.

### Pre-flight Modal (RECO-01, RECO-02 / D-01 through D-05)

**Trigger:** Single "Run Pipeline" button (replaces both "Run Cleaning" and "Run Harmonization")
- Button: `actionButton(ns("run_pipeline"), "Run Pipeline", class = "btn-success btn-lg mt-3 mb-3", icon = icon("play"))`
- Located at top of Clean Data tab and removed from Harmonize tab

**Modal structure:**
```
modalDialog(
  title = "Pre-flight Check",
  size = "m",
  easyClose = FALSE,
  footer = tagList(
    modalButton("Cancel"),
    actionButton(ns("run_checked"), "Run Checked Steps", class = "btn-primary"),
    actionButton(ns("run_all"), "Run All Steps", class = "btn-outline-secondary")
  )
)
```

**Checklist layout — two accordion sections:**
- Section 1: "Cleaning Steps" (accordion panel, open by default)
- Section 2: "Harmonization Steps" (accordion panel, open by default)

Each row within a section:
```
[ checkbox ] Step Name    [badge: "~N changes"] or [badge: "skip"]
```
- Firing steps: checkbox pre-checked, badge shows estimated change count (Bootstrap `badge bg-secondary`)
- Skipping steps (0 estimated changes): checkbox pre-unchecked, badge shows "skip" (Bootstrap `badge bg-light text-muted`)
- User can toggle any checkbox before running

**Zero-changes state (D-04):**
Skip modal entirely. Show `showNotification()`:
```
"Pre-flight: no steps have changes to apply. Run anyway?"
```
Notification type: `"message"`, duration: 0 (persistent until dismissed or run triggered).
Include an `actionLink` labeled "Open pre-flight modal" that shows the full modal.

### Media Classification Table (MEDIT-01, MEDIT-02, MEDIT-03 / D-06 through D-14)

**Placement:** New card/panel at bottom of Harmonize tab editors accordion.
Panel title: "Media Classification"

**Table rendering:** `DT::DTOutput()` (consistent with existing unmatched-unit DT usage in the module).

**Table columns (D-08):**
| Column | Display label | Width | Notes |
|--------|---------------|-------|-------|
| term | Term | auto | Raw media string, read-only display |
| canonical | Canonical | auto | Resolved ENVO term |
| source | Source | 80px | Badge-rendered (see below) |
| active | Active | 60px | Checkbox or Yes/No |

**Row grouping (D-07):**
- UNMATCHED section at top: rows where `canonical_media` is NA or flagged as unmatched.
  Highlight with Bootstrap `table-warning` row class (yellow-tinted).
  Section header row: "UNMATCHED — click a row to assign a canonical value"
- MAPPED section below: all successfully mapped rows.
  Standard alternating row style.

**Source badge rendering (D-12):**
- `source == "user"`: blue badge `<span class="badge bg-primary">user</span>` — row is editable
- `source == "amos"`: gray badge `<span class="badge bg-secondary">amos</span>` — row is read-only in editor

**Row click → edit modal (D-06):**
- Clicking any UNMATCHED row or any user-source row opens edit modal
- Clicking any AMOS-source row: no edit modal; show a brief notification: "AMOS entries are read-only. Add a user mapping for this term to override."

**Edit modal structure:**
```
modalDialog(
  title = "Edit Media Mapping",   # or "Add Media Mapping" for unmatched
  textInput(ns("modal_media_term"), "Term", value = row$term),
  textInput(ns("modal_media_canonical"), "Canonical", placeholder = "e.g., freshwater"),
  checkboxInput(ns("modal_media_active"), "Active", value = TRUE),
  footer = tagList(
    modalButton("Discard"),
    actionButton(ns("save_media_mapping"), "Save Mapping", class = "btn-primary")
  ),
  easyClose = FALSE
)
```

**AMOS override confirmation (D-13):**
When user saves a mapping for a term that already has an AMOS entry:
```
modalDialog(
  title = "Override AMOS Mapping?",
  p(sprintf('This term already has an AMOS mapping (canonical: "%s").', amos_canonical)),
  p("Your mapping will take priority at runtime. The AMOS entry will remain as fallback."),
  footer = tagList(
    modalButton("Cancel"),
    actionButton(ns("confirm_amos_override"), "Override", class = "btn-primary")
  )
)
```

**Re-run prompt after save (D-10):**
`showNotification()`:
```
"Media mappings updated. Re-run harmonization to apply changes?"
```
Include `actionLink` labeled "Re-run now" that triggers harmonization via `shinyjs::click()`.
Notification type: `"message"`, duration: 8 seconds.

## Interaction States

### "Run Pipeline" Button

| State | Appearance |
|-------|-----------|
| No data loaded / no tags | Disabled (`shinyjs::disabled()`) — same pattern as existing "Run Harmonization" |
| Ready to run | Enabled, `btn-success btn-lg` |
| Modal open | Button not re-clickable until modal is dismissed |
| Pipeline running | Button disabled, spinner shown (existing loading pattern) |

### Media Table Rows

| State | Appearance |
|-------|-----------|
| Unmatched row | `table-warning` Bootstrap row class (yellow tint) |
| AMOS-source row | `bg-secondary` badge, cursor default, no click modal |
| User-source row | `bg-primary` badge, cursor pointer, opens edit modal on click |
| Active = FALSE | Row text muted (`text-muted`) |

### Modals

All modals: `easyClose = FALSE` — user must explicitly Discard or Save. This matches the
existing unit mapping and correction modal pattern in mod_harmonize.R.
