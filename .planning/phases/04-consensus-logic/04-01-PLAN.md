<feature>
  <name>Consensus Classification</name>
  <files>R/consensus.R, tests/test_consensus.R</files>
  <behavior>
    classify_consensus(df, dtxsid_cols) takes a data frame (output of map_results_to_rows with multiple tagged columns) and a character vector of DTXSID column names, returns the df with added columns:
    - consensus_status: character ("agree", "agree_caveat", "disagree", "single", "error")
    - qc_tier: integer (1 = best to K = worst, where K = number of tagged columns)
    - consensus_dtxsid: character (the agreed DTXSID for agree/single rows, NA for disagree/error)
    - consensus_source: character ("consensus" if all agreed, column name if single, NA if disagree/error)

    Classification rules:
    - agree: All non-NA DTXSIDs across columns are identical (and at least 2 columns have values)
    - agree_caveat: Some columns have DTXSIDs that agree, but other columns are NA (at least 2 columns have same DTXSID, at least 1 is NA)
    - disagree: Two or more columns have different non-NA DTXSIDs
    - single: Exactly one column has a non-NA DTXSID (others are all NA)
    - error: All DTXSID columns are NA for that row

    compute_qc_tier(consensus_status, n_matched, n_total) returns integer QC tier:
    - Tier 1: All K columns agree (consensus_status == "agree")
    - Tier 2..K-2: Partial agreement (agree_caveat), rank by proportion matched
    - Tier K-1: Any disagreement (consensus_status == "disagree")
    - Tier K: No data (consensus_status == "error")
    - Single-source rows get tier based on position between agree_caveat and disagree

    Test cases:
    - 3 columns all same DTXSID -> agree, tier 1, consensus_dtxsid set
    - 2 of 3 columns agree, 1 NA -> agree_caveat, tier 2, consensus_dtxsid set
    - 2 columns different DTXSID -> disagree, tier K-1, consensus_dtxsid NA
    - 1 column has DTXSID, rest NA -> single, appropriate tier, consensus_dtxsid set
    - All columns NA -> error, tier K, consensus_dtxsid NA
    - 2 columns: both agree -> agree, tier 1
    - 2 columns: both different -> disagree, tier 1 (K-1 = 1 when K=2... recalculate)
    - Mixed rows in same df -> each classified independently
  </behavior>
  <implementation>
    Create R/consensus.R with:

    1. classify_consensus(df, dtxsid_cols):
       - For each row, extract DTXSIDs from the specified columns
       - Count non-NA values (n_present) and unique non-NA values (n_unique)
       - Apply classification rules:
         * n_present == 0 -> "error"
         * n_present == 1 -> "single"
         * n_unique == 1 and n_present == length(dtxsid_cols) -> "agree"
         * n_unique == 1 and n_present < length(dtxsid_cols) -> "agree_caveat"
         * n_unique > 1 -> "disagree"
       - Set consensus_dtxsid: the common DTXSID for agree/agree_caveat/single, NA for disagree/error
       - Set consensus_source: "consensus" for agree/agree_caveat, column name for single, NA for disagree/error
       - Compute QC tier via compute_qc_tier()

    2. compute_qc_tier(status, n_matched, n_total):
       - K = n_total (number of tagged columns)
       - agree (all K match): tier 1
       - agree_caveat: tier = 1 + (K - n_matched) — scales with how many are missing
       - single: tier = K - 1 (better than disagree, worse than any multi-match)
       - disagree: tier = K (active conflict is worst actionable state)
       - error: tier = K + 1 (no data at all, worst)
       Note: This means tiers range from 1 to K+1, not 1 to K. The context said "scaled to K" but K+1 tiers are needed to distinguish disagree from error. This is Claude's discretion per CONTEXT.md.

    3. Helper: find_dtxsid_cols(df) — auto-detect columns matching pattern "dtxsid_*" for convenience
  </implementation>
</feature>
