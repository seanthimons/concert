<feature>
  <name>Conflict Resolution</name>
  <files>R/consensus.R, tests/test_consensus.R</files>
  <behavior>
    get_resolution_options(df, row_idx, dtxsid_cols) returns a named list of available choices for a disagree row:
    - Only includes columns that have a non-NA DTXSID for that row
    - Returns list(column_name = dtxsid_value, ...) — the available choices
    - Returns empty list if row is not "disagree" status

    resolve_row(df, row_idx, chosen_column, dtxsid_cols) resolves a single disagreement row:
    - Sets consensus_dtxsid to the DTXSID from the chosen_column
    - Sets consensus_source to the chosen_column name
    - Sets .pinned = TRUE on that row (protects from future en masse changes)
    - Returns the modified df
    - Error if chosen_column doesn't have a DTXSID for that row
    - Error if row is not "disagree" status

    apply_priority_chain(df, priority_order, dtxsid_cols) applies en masse resolution:
    - priority_order is a character vector of column names ranked by preference (e.g., c("dtxsid_CAS", "dtxsid_Chemical"))
    - For each row with consensus_status == "disagree" AND .pinned != TRUE:
      - Walk priority_order, pick the first column that has a non-NA DTXSID
      - Set consensus_dtxsid and consensus_source accordingly
      - Do NOT set .pinned (en masse resolutions can be re-applied)
    - Rows with .pinned == TRUE are SKIPPED entirely
    - Returns the modified df

    init_resolution_state(df) adds .pinned = FALSE column if not present

    Test cases:
    - resolve_row on disagree row -> consensus_dtxsid filled, .pinned TRUE
    - resolve_row on non-disagree row -> error
    - resolve_row with invalid column -> error
    - apply_priority_chain fills all non-pinned disagree rows
    - apply_priority_chain skips pinned rows
    - apply_priority_chain with no disagree rows -> df unchanged
    - resolve_row then apply_priority_chain -> pinned row preserved
    - get_resolution_options returns only columns with data
    - get_resolution_options on non-disagree row -> empty list
    - Re-applying priority chain with different order updates non-pinned rows
  </behavior>
  <implementation>
    Extend R/consensus.R with:

    1. init_resolution_state(df):
       - Add .pinned = FALSE column if not present
       - Return df

    2. get_resolution_options(df, row_idx, dtxsid_cols):
       - Validate row_idx is valid
       - Check consensus_status == "disagree"
       - For each dtxsid_col, check if value is non-NA for this row
       - Return named list of column_name -> dtxsid_value for non-NA columns
       - Return empty named list() if not disagree

    3. resolve_row(df, row_idx, chosen_column, dtxsid_cols):
       - Validate: row exists, is "disagree", chosen_column has non-NA DTXSID
       - df <- init_resolution_state(df)
       - df[row_idx, "consensus_dtxsid"] <- df[row_idx, chosen_column]
       - df[row_idx, "consensus_source"] <- sub("^dtxsid_", "", chosen_column)
       - df[row_idx, ".pinned"] <- TRUE
       - Return df

    4. apply_priority_chain(df, priority_order, dtxsid_cols):
       - df <- init_resolution_state(df)
       - For each disagree row where .pinned != TRUE:
         * Walk priority_order
         * First column with non-NA DTXSID -> set consensus_dtxsid and consensus_source
       - Return df
  </implementation>
</feature>
