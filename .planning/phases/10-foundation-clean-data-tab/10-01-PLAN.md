    - Test: run_cleaning_pipeline on a 3-row dataframe with unicode and whitespace issues returns cleaned data + audit trail with correct row_id, field, step, original_value, new_value, reason columns
    - Test: run_cleaning_pipeline on clean data returns same data + empty audit trail (0 rows)
    - Test: audit trail only contains rows where original_value != new_value
  </behavior>
  <action>
    **First, write tests in tests/test_cleaning_pipeline.R** covering all behaviors above. Run them — they must fail (RED).

    **Then, create R/cleaning_pipeline.R** with these functions:

    1. `clean_unicode_field(x)` — Uses `stringi::stri_trans_general(x, "latin-ascii")` for unicode-to-ASCII transliteration. Handles NA values by returning NA. Do NOT use `iconv()` (drops characters instead of transliterating — see research Pitfall 1).

    2. `clean_text_field(x)` — Chain of: `stringr::str_trim()` → `stringr::str_squish()` → strip leading/trailing underscores and asterisks with `stringr::str_remove_all(x, "^[_*]+|[_*]+$")`. Do NOT strip all punctuation (breaks CAS numbers and IUPAC names — see research Pitfall 2). Only strip leading/trailing punctuation artifacts.

    3. `build_audit_trail(df_original, df_cleaned, step_name, reason_fn)` — Compares two dataframes column-by-column, row-by-row. Only records rows where `original_value != new_value`. Returns a tibble with columns: row_id (integer), field (character), step (character), original_value (character), new_value (character), reason (character). Use `as.character()` on both values for comparison safety.

    4. `run_cleaning_pipeline(df, reference_lists = NULL)` — Orchestrator function:
       - Step 1: Apply `clean_unicode_field()` across all character columns via `dplyr::mutate(across(where(is.character), clean_unicode_field))`
       - Step 2: Build unicode audit trail
       - Step 3: Apply `clean_text_field()` across all character columns
       - Step 4: Build trim audit trail
       - Step 5: Combine audit trails with `dplyr::bind_rows()`
       - Returns `list(cleaned_data = df_final, audit_trail = audit_combined)`
       - The `reference_lists` parameter is accepted but unused in Phase 10 (future phases will use it for stop word filtering etc.)

    Run tests — they must pass (GREEN).

    **Then, create R/cleaning_reference.R** with these functions:

    1. `load_or_fetch_reference(cache_path, fetch_fn, name)` — Generic cache-or-fetch function:
       - If `file.exists(cache_path)` is TRUE, `readRDS(cache_path)` and return
       - Otherwise, call `fetch_fn()`, create directory with `fs::dir_create(dirname(cache_path), recurse = TRUE)`, `saveRDS(result, cache_path, compress = FALSE)`, return result
       - Use `message()` for logging (not `showNotification` — this function may run outside Shiny context)

    2. `load_stop_words(cache_dir)` — Loads stop word list:
       - Cache path: `file.path(cache_dir, "stop_words.rds")`
       - fetch_fn: Returns a character vector of default chemistry stop words: c("test", "sample", "unknown", "blank", "standard", "control", "reference", "placeholder", "tbd", "tba", "na", "n/a", "none", "not available", "not applicable")
       - These are app defaults, not ComptoxR-seeded (stop words are domain knowledge)

    3. `load_block_patterns(cache_dir)` — Loads block pattern list:
       - Cache path: `file.path(cache_dir, "block_patterns.rds")`
       - fetch_fn: Returns character vector of regex patterns for substances to block from curation: c("^\\\\s*$", "^-+$", "^[.]+$", "^proprietary", "^confidential", "^trade\\\\s*secret", "^not\\\\s+disclosed")
       - These are app defaults — patterns matching empty/redacted/proprietary entries

    4. `load_functional_categories(cache_dir)` — Loads functional use categories:
       - Cache path: `file.path(cache_dir, "functional_categories.rds")`
       - fetch_fn: Wraps `tryCatch(ComptoxR::ct_functional_use("", domain = "func_use"), error = function(e) { message("ComptoxR unavailable, using empty functional categories"); tibble::tibble(name = character()) })`
       - Graceful fallback if ComptoxR API unavailable — app still starts

    5. `load_all_reference_lists(cache_dir)` — Convenience wrapper:
       - Calls all three loaders
       - Returns `list(stop_words = ..., block_patterns = ..., functional_categories = ...)`
