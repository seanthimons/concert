    SECTION 1 -- Output schema (DATE-02): Verify 5-column tibble with correct names and types.
    SECTION 2 -- Format families (DATE-01): One test_that block per format family: ISO ymd, YYYYMMDD compact, SAS dBY, MDY US, DMY European, named month-year (bY), numeric month-year (Ym).
    SECTION 3 -- Ambiguity flagging (DATE-03): "01/02/2015" is ambiguous; "13/02/2015" is NOT ambiguous.

    **Implementation** in `R/date_parser.R`:

    File header: `# date_parser.R` / `# Date string parsing engine: format detection, partial date handling, ambiguity flagging.` / `# Public API: parse_dates()`

    Roxygen block: `@param raw_dates Character vector of date strings`, `@param orig_row_id Integer vector of row IDs (default seq_along(raw_dates))`, `@return Tibble with 5 columns: orig_row_id, raw_date, parsed_date, date_year, date_flag`, `@importFrom tibble tibble`, `@export`

    Function body:
    1. Define constants at top of function: `ORDERS <- c("ymd", "Ymd", "bY", "BY", "dBY", "BdY", "mdy", "dmy", "Y", "Ym")` and `TWO_DIGIT_PAT <- "[0-9]{1,2}/[0-9]{1,2}/[0-9]{2}$"` per D-01/D-02.
    2. Empty-input guard: if `length(raw_dates) == 0`, return typed empty tibble with 5 columns (same pattern as unit_harmonizer.R lines 302-312).
    3. Vectorized parse: `parsed_posix <- lubridate::parse_date_time(raw_dates, orders = ORDERS, train = FALSE, quiet = TRUE)` -- **train=FALSE is critical** per Pitfall 1.
    4. Format output: `parsed_date <- format(as.Date(parsed_posix), "%Y-%m-%d")` then set NA where parsed_posix is NA.
    5. Extract year: `date_year <- lubridate::year(parsed_posix)` then set NA_integer_ where parsed_posix is NA.
    6. Flag assignment using `dplyr::case_when()` for priority enforcement (unparseable > partial > inferred_format > ambiguous > ""):
       - `is_unparseable`: `is.na(parsed_posix)`
       - `is_partial`: `!is_unparseable & (grepl("^[0-9]{4}$", trimws(raw_dates)) | grepl("^[0-9]{4}[-/][0-9]{1,2}$", trimws(raw_dates)) | grepl("^[A-Za-z]+ [0-9]{4}$", trimws(raw_dates)))` -- Pitfall 4: partial check must precede ambiguity to avoid "2015" false-positive.
       - `is_inferred`: `!is_unparseable & grepl(TWO_DIGIT_PAT, raw_dates, perl = TRUE)` per D-09.
       - `is_ambiguous`: `!is_unparseable & !is_partial & lubridate::day(parsed_posix) <= 12 & lubridate::month(parsed_posix) <= 12` per DATE-03. Note the `!is_partial` guard (Pitfall 4).
    7. Return tibble: `tibble::tibble(orig_row_id = as.integer(orig_row_id), raw_date = as.character(raw_dates), parsed_date, date_year = as.integer(date_year), date_flag)`.

    Code comment on cutoff: `# 2-digit year cutoff: lubridate default (year < 69 -> 2000+year, >= 69 -> 1900+year). Acceptable for regulatory data spanning 1950-2030. Per D-08, this is the selected threshold.`

    After implementation, run `air format R/date_parser.R` and `jarl check R/date_parser.R`.
