## Spacing Scale

The app uses Bootstrap 5 utility classes. Declared equivalents for design decisions in this phase:

| Token | Bootstrap utility | px equivalent | Usage |
|-------|------------------|---------------|-------|
| xs | gap/padding helper | 4px | Icon gaps, inline badge padding |
| sm | mb-2, me-1 | 8px | Chip-to-chip margin, row item gaps |
| md | mb-3, mt-3, py-3 | 16px | Section spacing within accordion panels |
| lg | py-4 | 24px | Accordion panel body padding |
| xl | py-5 | 32px–48px | Empty state vertical centering |
| modal-form | mb-3 per field | 16px | Modal form field separation |

Exceptions:
- Unmatched unit row list items: `mb-2` (8px) between each unit+action row — tight list, not loose sections

- Chip container: `max-height: 200px; overflow-y: auto; padding: 6px` — reused directly from `mod_clean_data.R` lines 20-21

**Source:** Existing modules — `mod_clean_data.R` and `mod_run_curation.R` spacing patterns.

### Chip Label Format

Unit mapping chips: `"{from_unit} → {to_unit}"` with optional multiplier suffix `" (×{multiplier})"`.
Example: `"ug/L → mg/L (×0.001)"`.

Corrections chips: `"{pattern} → {replacement}"`.
Example: `"1.5E 3 → 1.5E3"`.

**Source:** `34-CONTEXT.md` specifics block, chip label spec.

## Interaction Contracts

### Pipeline Execution (UITG-04)

1. Tab renders with "Run Harmonization" button (enabled when `data_store$numeric_tags` is non-null).
2. If `numeric_tags` is null: show empty state — `div(class = "text-center text-muted py-5")` with `bs_icon("sliders", size = "3em")`, heading "No numeric columns tagged", body "Tag your Result and Unit columns first, then run harmonization."
3. Button click triggers `withProgress()` pipeline with at minimum 2 stages: "Parsing numeric results..." and "Harmonizing units...".
4. On completion: write `data_store$harmonize_results` and `data_store$harmonize_audit`. Render value boxes. Accordion panels become interactive.
5. Re-run: button remains active after completion for re-runs after editor changes.

### Accordion Behavior (D-04, D-05)

- All three accordion panels start collapsed (`open = FALSE`).
- Panels are independently openable; no "open one at a time" constraint.
- Panel titles include item count badge: "Unit Table Editor (151 mappings)", "Corrections Editor (0 corrections)", "Unmatched Units (N unmatched)".

### Chip Editor Interactions (D-06 through D-11)

- Click chip body → open edit modal pre-filled with all fields.
- Click chip × button → remove immediately, no confirmation modal (single items only; non-default items only).
- "Add Unit Mapping" button → open blank add modal.
- Modal "Save" → append/update chip, trigger cascade reset (D-26).
- App-default unit mappings: no × button (not removable).

### Unmatched Unit Panel (D-12 through D-16)

- Panel title: "Unmatched Units (N)" where N is count from last harmonization run.
- If harmonization not yet run: panel shows "Run harmonization to see unmatched units."
- After run: list of rows formatted as `"{unit_string} ({row_count} rows)"` with "Add Mapping" button aligned right (`d-flex justify-content-between align-items-center mb-2`).
- "Add All as Pass-through" button: adds all unmatched units as identity mappings (from_unit = to_unit, multiplier = 1, category = "dimensionless", confidence = "LOW", source = "user_passthrough"). Placed above the list with `hr()` separator.
- "Add Mapping" per row: opens unit mapping modal pre-filled with `from_unit` = that unit string.
- After adding mappings, user re-runs harmonization to apply (D-16) — no automatic re-run.

### Cascade Reset (D-26, D-27, D-28)

- Unit table or corrections table change → null `data_store$harmonize_results` and `data_store$harmonize_audit`. Value boxes disappear. Unmatched panel reverts to "Run harmonization to see unmatched units."
- Harmonization results change → null `data_store$toxval_output` (downstream Phase 35 reset).

## State Visibility Rules

| UI state | Condition | What is shown |
|----------|-----------|---------------|
| No numeric tags | `is.null(data_store$numeric_tags)` | Empty state only |
| Tags present, not run | `!is.null(numeric_tags) && is.null(harmonize_results)` | Button only |
| Pipeline running | `withProgress()` active | Button disabled, progress bar |
| Run complete | `!is.null(harmonize_results)` | Button + value boxes + accordions |
| Editor changed post-run | cascade reset fires | Value boxes disappear, button remains |

**Source:** `34-CONTEXT.md` D-03, D-05, D-21 through D-28; pattern from `mod_run_curation.R` conditionalPanel usage.
