## Spacing Scale

Standard Bootstrap 5 spacing utilities apply throughout. Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding (e.g., `px-1` / `py-0`) |
| sm | 8px | Compact element spacing, badge internal padding (`px-2`) |
| md | 16px | Default card body padding, modal content gap |
| lg | 24px | Section padding between value boxes and table |
| xl | 32px | Layout gaps between major UI regions |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Candidate cards in comparison modal: `mb-2` (8px) between cards, consistent with existing pattern at mod_review_results.R L1259
- Status chip font-size: 0.85em (matches existing chip rendering at L889)
- Bulk accept button positioned in the existing action button row at the top of Curated Results; height matches `btn-sm` siblings (32px)

## Interaction Contracts

### Status Chip Filter

The `consensus_status` dropdown filter (select input above the reactable column) MUST include `"auto_resolved"` and `"suggested"` as selectable values whenever those statuses are present in the displayed data. Filter behavior is exact-match (existing JS filter function unchanged).

### Resolution Column — auto_resolved row

Display: `"✅ {dtxsid} — {preferredName} " + '<span class="badge" style="background:#0D6EFD;...">auto</span>'`
Compare button present (per D-14 override flow): `btn btn-sm btn-outline-primary` with search icon + "Compare" label.

### Resolution Column — suggested row

Display: `'<button class="compare-btn btn btn-sm btn-outline-info" data-row="{n}">🔍 Review Suggestion</button>'`
The button color `btn-outline-info` (cyan) differentiates from plain DISAGREE rows which use `btn-outline-primary`.

### Comparison Modal — suggested row

When modal opens for a `suggested` row:
1. Suggested candidate card has `border: 2px solid #0D6EFD` (blue highlight border, not default `#dee2e6`).
2. A `badge bg-primary` label "Suggested" appears next to the DTXSID heading inside that card.
3. An "Accept Suggestion" `btn btn-primary` button appears in the modal footer alongside "Confirm & Close" and "Skip this row".
4. Clicking "Accept Suggestion" selects the suggested candidate and resolves the row; sets `.resolution_method = "suggested-accept"`.
5. User may still click "Select" on any other card to override the suggestion manually; that path sets `.resolution_method = "manual"`.

### Comparison Modal — auto_resolved row (override)

When modal opens for an `auto_resolved` row (per D-14):
1. Auto-resolved candidate card has `border: 2px solid #FFC107` (warning amber) to signal "currently auto-selected".
2. A `badge bg-warning text-dark` label "Auto-Selected" appears inside that card.
3. No "Accept Suggestion" button (already resolved). Footer shows only "Confirm & Close" and "Skip this row".
4. User picks a different candidate → sets `.resolution_method = "manual"`.

### Bulk Accept All Suggestions Button

Placement: In the existing `d-flex gap-2` action button row (mod_review_results.R L389), after "Apply Priority" and before "Show Errors". Hidden via `shinyjs::hidden()` when `n_suggested == 0`.

Revealed state: `shinyjs::show()` when `data_store$resolution_state` contains any row with `consensus_status == "suggested"`.

Label: "Accept All Suggestions"
Icon: `icon("wand-magic-sparkles")` (Font Awesome 6)
Class: `btn-sm btn-primary`

Confirmation: No modal confirmation required (per D-10; threshold decision already made at classification). Action applies immediately. Shows `showNotification("{n} rows auto-accepted", type = "message")` on completion.

Skips pinned rows (consistent with `apply_priority_chain()` pattern from D-09).

### .resolution_reason Display

The `.resolution_reason` column (e.g., `"score=0.98, gap=0.45, threshold=0.95"`) is surfaced only in:
1. The exported Excel `.resolution_reason` column (no UI change needed — flows through existing export).
2. Optionally as a `title` tooltip on the AUTO chip in the Resolution column: `title="score=0.98, gap=0.45"`.

It is NOT rendered as visible body text in the table row to preserve column width. Tooltip only.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (modal, suggested row) | "Accept Suggestion" |
| Primary CTA (bulk action) | "Accept All Suggestions" |
| Auto-resolved chip label | `auto_resolved` (matches R value; lowercase with underscore, consistent with existing chip rendering) |
| Suggested chip label | `suggested` (matches R value) |
| Suggested row button label | "Review Suggestion" |
| Auto-resolved candidate badge | "Auto-Selected" |
| Suggested candidate badge | "Suggested" |
| Value box title (auto) | "Auto-Resolved" |
| Value box title (suggest) | "Suggested" |
| Bulk accept notification | "{n} suggestion(s) accepted" (e.g., "12 suggestion(s) accepted") |
| Empty state — no suggestions | Not shown. When `n_suggested == 0`, "Accept All Suggestions" button is hidden; no empty state message needed. |
| Error state — bulk accept fails | "Failed to accept suggestions. Please try again." (`showNotification(..., type = "error")`) |
| Override confirmation | None. Picking a different candidate in the modal is self-confirming via the existing "Confirm & Close" flow. |
| Reject suggestion note | No "rejected" state per D-15. Row stays `suggested`. No copy needed. |
