       - Input: character vector of unit strings

    2. `harmonize_units(values, units, unit_map)` (exported):
       - Input:
         - `values`: numeric vector of parsed numeric values
         - `units`: character vector of unit strings (same length as values)
         - `unit_map`: tibble from load_unit_map() with columns from_unit, to_unit, multiplier
       - Step 1: Capture `orig_unit = units` before any transformation
       - Step 2: Assign `orig_row_id = seq_along(values)`
       - Step 3: Normalize units via `normalize_unit_string(units)`
       - Step 4: For each row, perform lookup:
         (a) Case-sensitive match: `normalized[i] == unit_map$from_unit`
         (b) If match found: use to_unit, multiplier from matched row; unit_flag = ""
         (c) If no match: case-insensitive fallback: `tolower(normalized[i]) == tolower(unit_map$from_unit)`
         (d) If fallback match found: use to_unit, multiplier; unit_flag = "case_fallback"
         (e) If still no match (per D-01): harmonized_value = values[i], harmonized_unit = orig_unit[i], conversion_factor = 1, unit_flag = "unmatched"
       - Step 5: Compute harmonized_value = values * conversion_factor
       - Step 6: Build output tibble with columns in exact order (per D-07):
         `orig_row_id`, `orig_unit`, `harmonized_value`, `harmonized_unit`, `conversion_factor`, `unit_flag`
       - Return: tibble
