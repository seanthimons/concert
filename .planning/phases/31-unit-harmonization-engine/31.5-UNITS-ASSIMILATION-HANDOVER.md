# Phase 31.5: Units Package Assimilation — Handover Document

**Created:** 2026-04-15
**Context:** Research from Phase 31 discuss-phase session
**Purpose:** Replace manual 151-row unit conversion table with `units` package + registrations

## Problem Statement

The current unit conversion table (`inst/extdata/unit_conversion.rds`) requires manual maintenance:
- 151 rows with explicit multipliers
- Multiple rows for spelling variants (µg/L, μg/L, ug/L)
- Adding new units requires calculating conversion factors
- No dimensional safety (can accidentally convert mg/L to mg/kg)

## Proposed Architecture

```
┌─────────────────────────────────────────────────────┐
│  units package (udunits2 engine)                    │
│  - 276 base units with dimensional algebra          │
│  - Automatic conversion factors                     │
├─────────────────────────────────────────────────────┤
│  + UCUM registrations (~30 units)                   │
│    - Molarity: M, mM, µM, nM, pM                    │
│    - Healthcare units not in udunits2               │
├─────────────────────────────────────────────────────┤
│  + Domain registrations (~20 units)                 │
│    - Turbidity: NTU, FTU, JTU                       │
│    - Microbial: CFU, MPN                            │
│    - Hardness: as CaCO3                             │
│    - Dose qualifiers: bw/day variants               │
├─────────────────────────────────────────────────────┤
│  + Synonym normalization table (~40 rows)           │
│    - µg/L, μg/L, ug/L → ug/L (before lookup)        │
│    - ppb → ug/L (context: aqueous)                  │
│    - mg/kg bw/day → mg/kg/d (normalized form)       │
└─────────────────────────────────────────────────────┘
```

### Custom Unit Registration (Proven Working)

```r
library(units)

# Molarity - moles per liter
install_unit("M", "mol/L", "molar")
install_unit("mM", "mmol/L", "millimolar")

# Turbidity - dimensionless
install_unit("NTU", name = "nephelometric turbidity unit")

# Test
x <- set_units(1, "M")
y <- set_units(x, "mM")
# Result: 1000 mM ✓

x <- set_units(5, "NTU")
# Result: 5 NTU ✓
```

harmonize_unit <- function(value, from_unit, to_unit = NULL) {
  # 1. Normalize input unit string
  normalized <- normalize_unit_string(from_unit)  # uses synonym table


  # 2. Set units
  x <- tryCatch(
    units::set_units(value, normalized, mode = "standard"),
    error = function(e) {
      # Unrecognized unit - pass through with flag
      list(value = value, unit = from_unit, flag = "unmatched")
    }
  )

  # 3. Convert if target specified

  if (!is.null(to_unit) && !inherits(x, "list")) {

## Risks & Mitigations

1. **`units` package dependency** — Already installed on target systems; add to Suggests with graceful fallback
2. **Registration order** — Some units depend on others; test load order
3. **ppb/ppm ambiguity** — These are dimensionless; require context (aqueous = mg/L, solid = mg/kg)
4. **String normalization quirks** — `mg/kg/day` becomes `mg/d/kg`; may need post-processing for display

## Additional Scope for 31.5 (from Phase 31 discussion)

### Context-Aware Conversions

**Problem identified:** Current table conflates air/water concentrations and has broken molarity conversions.

**Issues to fix:**
1. `mg/m³ → mg/L` is WRONG for air — air concentrations should stay as `mg/m³`
2. `mol/L → mg/L (x1.0)` is WRONG — needs molecular weight
3. `ppm/ppb` appears in both concentration and mass_fraction — needs media context

**Decisions (from user):**

1. **Molarity handling:** Fetch MW via CompTox API when molarity unit detected
   ```r
   # Conversion formula
   mg_L = molarity_M * molecular_weight_g_mol * 1000

   # For mM: mg/L = mM × MW
   # For µM: mg/L = µM × MW × 0.001
   ```

2. **Media inference:** Derive from column tags
   - `exposure_route = "inhalation"` → media = "air" → target = `mg/m³`
   - `exposure_route = "oral"/"dermal"` or aquatic context → media = "aqueous" → target = `mg/L`
   - Solid matrix → media = "solid" → target = `mg/kg`

3. **ppm/ppb routing:** Use inferred media to pick correct conversion path

**Expanded interface:**
```r
harmonize_unit(

value,
  from_unit,
  media = NULL,           # "aqueous" | "air" | "solid" — inferred from tags
  dtxsid = NULL,          # for MW lookup when molarity detected
  molecular_weight = NULL # optional override, skips API call
)
```

### Table Fixes Required

| Current | Problem | Fix |
|---------|---------|-----|
| `mg/m³ → mg/L` | Conflates air/water | New `air_concentration` category → `mg/m³` |
| `mol/L → mg/L (x1.0)` | Missing MW | Flag `needs_mw`, fetch via API |
| `M, mM, µM, nM, pM → mg/L (x1.0)` | Missing MW | Same — all molarity units need MW |
| `ppm → mg/L` AND `ppm → mg/kg` | Ambiguous | Route based on media context |
