# Phase 31.5 Plan 03: Context-Aware Unit Harmonization Summary

**One-liner:** Upgraded harmonize_units() with molarity-to-mg/L conversion (via MW lookup), ppb/ppm media-based routing, and internal synonym normalization while maintaining backward compatibility.

### 2. Upgraded harmonize_units() (R/unit_harmonizer.R)

Extended signature with optional context parameters (backward compatible):

```r
harmonize_units(
  values,
  units,
  unit_map,
  media = NULL,           # "aqueous" | "air" | "solid" | NULL
  dtxsid = NULL,          # for MW lookup when molarity detected
  molecular_weight = NULL # optional override, skips API call
)
```

**New features:**

**Synonym normalization (internal):**
- Loads synonyms via system.file() at function start
- Applies before table lookup
- Normalizes variants like "mg/kg bw/day" -> "mg/kg/d"

**Molarity detection and conversion:**
- Detects M, mM, uM, nM, pM, mol/L, mmol/L, etc.
- If MW available (param or API), converts to mg/L using formula:
  - `mg/L = molarity * MW * scale_factor`
  - Scale factors: M=1000, mM=1, uM=0.001, nM=1e-6, pM=1e-9
- If MW unavailable, sets unit_flag="needs_mw" and passes through

**ppb/ppm media routing:**
- Routes based on media context:
  - aqueous -> mg/L
  - solid -> mg/kg
  - air -> mg/m3
- NULL media defaults to aqueous with unit_flag="media_inferred"

**Extended unit_flag values:**
- "" = exact match, successful conversion
- "case_fallback" = case-insensitive match
- "unmatched" = no match, passed through
- "needs_mw" = molarity unit but no MW available
- "media_inferred" = ppb/ppm converted using default aqueous

**Helper functions added:**
- `get_unit_synonyms()` - internal synonym loader
- `apply_synonyms()` - applies synonym normalization
- `is_molarity_unit()` - detects molarity units
- `get_molarity_scale()` - returns scale factor for unit
- `fetch_molecular_weight()` - ComptoxR bulk API lookup
- `get_media_target()` - returns target unit for ppb/ppm
- `get_ppx_conversion_factor()` - returns 0.001 for ppb, 1 for ppm
