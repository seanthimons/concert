<design_decision>
## Internal Script Rationale

Per plan-checker feedback: unit_extraction.R uses DBI, duckdb, and arrow but these are
NOT added to DESCRIPTION Imports. This is intentional:

1. Extraction functions are INTERNAL (not exported) - one-time data build step
2. Users never call these functions - they consume the pre-built .rds files
3. Adding 3 heavy dependencies (DBI ~2MB, duckdb ~50MB, arrow ~30MB) for internal-only
   functions bloats the package unnecessarily
4. Functions are marked @keywords internal and documented as requiring manual
   dependency installation when run

If someone wants to rebuild the conversion table, they install duckdb/arrow manually.
The shipped package only needs the resulting .rds files.
</design_decision>
</context>

  <name>Task 1: Create internal unit extraction module</name>
  <files>R/unit_extraction.R</files>
  <action>
Create R/unit_extraction.R with four INTERNAL functions (NOT exported):

1. `extract_ecotox_units(db_path)` - Query ECOTOX ecotox.duckdb:
   - Connect via DBI::dbConnect(duckdb::duckdb(), db_path, read_only=TRUE)
   - Query unit_conversion table: SELECT orig, cur_unit_result, multiplier
   - Transform SI base targets to UCUM: g/l -> mg/L (multiply by 1000), mol/l -> M (keep), etc.
   - Return tibble with: from_unit, to_unit, multiplier, category, confidence, source
   - **Per D-15:** Add category inference that distinguishes:
     * `concentration` (aqueous): target = mg/L
     * `air_concentration`: target = mg/m3 (NOT mg/L)
     * `mass_fraction`: target = mg/kg
     * `dose`: target = mg/kg/d
     * `molarity`: confidence = NEEDS_MW (no numeric multiplier)
   - **Per D-14:** Mark molarity entries (mol/l, M, mM, uM, nM, pM) with:
     * confidence = "NEEDS_MW"
     * multiplier = NA_real_ (NOT a numeric value like 1.0)

2. `extract_toxval_units(db_path)` - Query ToxVal toxval.duckdb:
   - SELECT DISTINCT toxval_units FROM toxval WHERE toxval_units IS NOT NULL
   - Return tibble with distinct unit strings for gap analysis
   - Do NOT create conversions - just inventory for coverage checking

3. `extract_sswqs_units(parquet_path)` - Read SSWQS parquet:
   - Use arrow::read_parquet() to read file
   - Extract distinct(unit) column values
   - Return tibble with distinct unit strings for gap analysis

4. `build_unit_conversion_table(ecotox_path, toxval_path = NULL, sswqs_path = NULL, output_path)`:
   - Call extract_ecotox_units() as foundation
   - If toxval_path provided, call extract_toxval_units() for gap analysis
   - If sswqs_path provided, call extract_sswqs_units() for gap analysis
   - Log coverage stats: N units covered, M units missing
   - Transform ECOTOX SI targets to UCUM conventions per D-02d:
     * g/l -> mg/L (multiply original multiplier by 1000)
     * mol/l -> M (keep multiplier NA, mark NEEDS_MW)
     * g/kg -> mg/kg (multiply by 1000)
     * g/m3 -> mg/m3 (multiply by 1000) for air_concentration
   - Deduplicate by from_unit (keep first occurrence)
   - Save to output_path via saveRDS()
   - Return the tibble

Include mappings (per RESEARCH.md findings):

Micro symbol variants:
- "ug/L", "ug/l" (already ASCII) -> no mapping needed (handled by normalize_unit_string)
- micro sign variants handled by existing normalize_unit_string()

Case normalization (for units not in table):
- Liter variants: l -> L (when lowercase l appears as unit)
- This is tricky - L/l distinction matters in some contexts

Body weight qualifiers (D-02d):
- "mg/kg bw/day" -> "mg/kg/d"
- "mg/kg-bw/day" -> "mg/kg/d"
- "ug/kg bw/day" -> "ug/kg/d"
- "g/kg bw/day" -> "g/kg/d"

Percent variants:
- "% (w/v)" -> "%w/v"
- "% (v/v)" -> "%v/v"
- "% (w/w)" -> "%w/w"
- "% w/v" -> "%w/v"
- "% v/v" -> "%v/v"
- "% w/w" -> "%w/w"

Dimensionless standardization:
- "ratio" -> "1" (or keep as-is, document)
- "fraction" -> "1"
- "unitless" -> "1"
- "dimensionless" -> "1"
- "none" -> "1"
- "-" -> "1"
