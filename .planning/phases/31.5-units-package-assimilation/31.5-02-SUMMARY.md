# Phase 31.5 Plan 02: Domain Unit Registrations Summary

**One-liner:** Registered molarity (M/mM/uM/nM/pM), turbidity (NTU/FTU/JTU), and microbial (CFU/MPN) units via .onLoad() hook with reload resilience.

## What Was Built

### 1. Package Dependency (DESCRIPTION)

Added `units (>= 0.8-0)` to Imports section. Version constraint ensures `install_unit()` is available (added in 0.8-0).

### 2. Unit Registrations (R/zzz.R)

Created `.onLoad()` hook that registers domain-specific units on package load:

**Molarity (based on mol/L):**
- M (molar) = 1 mol/L
- mM (millimolar) = 0.001 mol/L
- uM (micromolar) = 0.000001 mol/L
- nM (nanomolar) = 10^-9 mol/L
- pM (picomolar) = 10^-12 mol/L

**Turbidity (dimensionless, not interconvertible):**
- NTU (nephelometric turbidity unit)
- FTU (formazin turbidity unit)
- JTU (jackson turbidity unit)

**Microbial (dimensionless counts):**
- CFU (colony forming unit)
- MPN (most probable number)

Key design decisions:
- No `requireNamespace()` guard per D-03 (units is hard Imports)
- `tryCatch()` around each registration block for reload resilience
- `register_chemreg_units()` is internal (not exported)

### 3. Tests (test-unit-registrations.R)

17 tests covering:
- Molarity unit recognition (5 tests: M, mM, uM, nM, pM)
- Molarity conversion chain (4 tests: M->mM, mM->uM, uM->nM, nM->pM)
- M <-> mol/L interoperability (1 test)
- Turbidity recognition (3 tests: NTU, FTU, JTU)
- Microbial recognition (2 tests: CFU, MPN)
- Built-in udunits2 compatibility (2 tests: mg/L, ug/L->mg/L)

No `skip_if_not_installed("units")` per D-03.
