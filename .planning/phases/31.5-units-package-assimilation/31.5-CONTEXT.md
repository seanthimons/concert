# Phase 31.5: Units Package Assimilation - Context

**Gathered:** 2026-04-15
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the minimal 4-row unit conversion table with a comprehensive system: extract units from ECOTOX/SSWQS/ToxValDB sources, integrate the R `units` package for automatic conversions, and add context-aware logic for molarity (MW lookup) and media-dependent units (ppb/ppm routing).

**Scope:**
1. **Data extraction:** Pull distinct units from local ECOTOX dump, SSWQS benchmark files, and ToxValDB schema
2. **Table curation:** Categorize, deduplicate, and fix data quality issues (broken molarity multipliers, air/water conflation)
3. **Units package integration:** Use udunits2 engine for automatic conversions where supported (~70% coverage)
4. **Domain registrations:** Register chemistry-specific units not in udunits2 (molarity, turbidity, microbial, hardness)
5. **Synonym normalization:** Map variant spellings to canonical forms before lookup
6. **Context-aware conversions:** Molarity→mg/L with MW, media-based ppb/ppm routing

**Not in scope:** Automatic unit inference from column names (anti-feature per PROJECT.md), bidirectional conversion, UCUM XML parsing.

</domain>

### Canonical Unit Format
- **D-02d:** UCUM conventions for canonical output symbols:
  - Aqueous concentration: `mg/L` (capital L per UCUM/SI)
  - Dose rate: `mg/kg/d` or `mg/kg-day` (TBD — check ToxVal convention)
  - Air concentration: `mg/m3`
  - Matrix concentration: `mg/kg`
- **D-02e:** ECOTOX table is foundation — lift 706 conversions, transform targets from SI base (g/l) to UCUM (mg/L)

### Package Integration
- **D-03:** `units` package as hard dependency (Imports, not Suggests). Lightweight package (~1MB), no external system deps, avoids fallback complexity.
- **D-04:** Register domain-specific units on package load via `.onLoad()` in `R/zzz.R`:
  - Molarity: M, mM, µM, nM, pM (based on mol/L)
  - Turbidity: NTU, FTU, JTU (dimensionless)
  - Microbial: CFU, MPN (dimensionless counts)
  - Hardness: as CaCO3 equivalent
  - Time shortcuts: wk, mo (if not in udunits2)

### Molecular Weight Lookup
- **D-05:** Live CompTox API lookup for MW when molarity unit detected AND DTXSID available in row context. Uses existing `ComptoxR::ct_chemical_detail_search_bulk()` pattern from Phase 17 enrichment.
- **D-06:** Conversion formula: `mg/L = molarity × MW × scale_factor`
  - M → mg/L: multiply by MW × 1000
  - mM → mg/L: multiply by MW
  - µM → mg/L: multiply by MW × 0.001
- **D-07:** If DTXSID unavailable or API fails, flag as `"needs_mw"` and pass through unconverted.

### Media Inference
- **D-08:** Media context derived from column tags when available:
  - `exposure_route = "inhalation"` → media = "air" → target = mg/m³
  - `exposure_route = "oral"/"dermal"` or aquatic context → media = "aqueous" → target = mg/L
  - Solid matrix indicators → media = "solid" → target = mg/kg
- **D-09:** Default to aqueous when media context unknown. Add `media_inferred = TRUE` to audit trail.
- **D-10:** ppb/ppm routing: use inferred media to select conversion path (aqueous = mg/L, solid = mg/kg, air = mg/m³).

### API Design
- **D-11:** Replace `harmonize_units()` in-place — same core signature, smarter implementation using `units` package internally.
- **D-12:** Extend signature with optional context params:
  ```r
  harmonize_units(
    values,
    units,
    unit_map,
    media = NULL,           # "aqueous" | "air" | "solid" | NULL (infer)
    dtxsid = NULL,          # for MW lookup when molarity detected
    molecular_weight = NULL # optional override, skips API call
  )
  ```
- **D-13:** Backward compatible: existing calls without new params continue to work (aqueous default, no MW conversion).

### Data Quality Fixes
- **D-14:** Remove/fix broken molarity entries: `mol/L, M, mM, µM → mg/L (×1.0)` is wrong — needs MW.
- **D-15:** Add `air_concentration` category separate from aqueous concentration. Target = mg/m³, not mg/L.
- **D-16:** Resolve dimensionless inconsistency: standardize on single canonical form for ratio/fraction/unitless.

### Audit Trail
- **D-17:** Extended unit_flag values:
  - `""` — exact match, successful conversion
  - `"case_fallback"` — case-insensitive match
  - `"unmatched"` — no match, passed through
  - `"needs_mw"` — molarity unit but no MW available
  - `"media_inferred"` — ppb/ppm converted using default aqueous assumption

## Specific Ideas

### Registration Pattern (from research)
```r
# R/zzz.R
.onLoad <- function(libname, pkgname) {
  if (requireNamespace("units", quietly = TRUE)) {
    register_chemreg_units()
  }
}

register_chemreg_units <- function() {
  # Molarity (based on mol/L)
  units::install_unit("M", "mol/L", "molar")
  units::install_unit("mM", "mmol/L", "millimolar")
  units::install_unit("uM", "umol/L", "micromolar")
  units::install_unit("nM", "nmol/L", "nanomolar")
  units::install_unit("pM", "pmol/L", "picomolar")

  # Turbidity (dimensionless)
  units::install_unit("NTU", name = "nephelometric turbidity unit")
  # ... etc
}
```
