    **Flag Matching (test_flag_matching.R):**
    - flag_reference_matches(df, name_cols, reference_list, flag_type, flag_label) returns list(cleaned_data, audit_trail)
    - Exact match: "plasticizer" matches reference term "plasticizer" -> flag with [exact]
    - Substring match: "dibutyl phthalate plasticizer" matches "plasticizer" -> flag with [substring]
    - Exact match takes priority over substring (if name IS the term exactly, label as exact)
    - Case-insensitive matching (both exact and substring)
    - Only active=TRUE entries are matched against
    - active=FALSE entries are skipped (soft delete)
    - flag_type="warning" produces "WARN: {flag_label} [match_type]"
    - flag_type="blocking" produces "BLOCK: {flag_label} [match_type]"
    - Audit trail records matched_term, match source (comptoxr/user/app_default), and match type
    - No flag if no match found
    - Handles NA name values without error

    **Part C: Add flag_reference_matches() to R/cleaning_pipeline.R**

    Two-pass matching per user decision:
    1. Pass 1 (exact): `tolower(name_value) == tolower(term)` for each active reference entry
    2. Pass 2 (substring): `stringr::str_detect(name_value, stringr::regex(term, ignore_case = TRUE))` only if no exact match

    For matched rows:
    1. Set `cleaning_flag` to `"{BLOCK|WARN}: {flag_label} [{exact|substring}]"`
    2. Record in audit trail: matched term, source, match type in reason field

    IMPORTANT: If a row already has a cleaning_flag (e.g., from bare formula detection), do NOT overwrite — first flag wins (bare formula blocking is higher priority than reference list warnings).
