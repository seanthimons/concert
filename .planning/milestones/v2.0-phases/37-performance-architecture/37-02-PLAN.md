**Also add a shared helper for constructing skip results:**

```r
#' Construct a pre-check skip result for a cleaning pipeline step
#'
#' @param df Dataframe to pass through unchanged.
#' @param step_name Character. The step being skipped (for message).
#' @return list with cleaned_data passthrough and empty typed audit trail.
#' @keywords internal
build_skip_result <- function(df, step_name) {
  message(sprintf("Step %s skipped -- pre-check FALSE", step_name))
  list(
    cleaned_data = df,
    audit_trail = tibble::tibble(
      row_id = integer(), field = character(), step = character(),
      original_value = character(), new_value = character(), reason = character()
    )
  )
}
```

**Pre-check functions (one per dedup-eligible step):**

1. **`precheck_unicode_to_ascii(df)`**
   - Extract all character columns: `char_cols <- names(df)[vapply(df, is.character, logical(1))]`
   - If `length(char_cols) == 0`, return `list(should_run = FALSE, est_changes = 0L)`
   - For each char col, test `stringi::stri_enc_isascii()` with `na.rm = TRUE`
   - `should_run <- !all(vapply(char_cols, function(col) all(stringi::stri_enc_isascii(df[[col]]), na.rm = TRUE), logical(1)))`
   - `est_changes`: sum of `sum(!stringi::stri_enc_isascii(df[[col]]), na.rm = TRUE)` across all char cols
   - Return `list(should_run = should_run, est_changes = as.integer(est_changes))`

2. **`precheck_trim_whitespace(df)`**
   - Extract all character columns
   - If `length(char_cols) == 0`, return `list(should_run = FALSE, est_changes = 0L)`
   - For each char col, check if `any(df[[col]] != clean_text_field(df[[col]]), na.rm = TRUE)`
   - `should_run` if any column has any difference
   - `est_changes`: total count of differing cells
   - Return `list(should_run = should_run, est_changes = as.integer(est_changes))`

3. **`precheck_normalize_cas(df, tag_map)`**
   - `cas_cols <- names(tag_map)[tag_map == "CASRN"]`
   - If `length(cas_cols) == 0`, return `list(should_run = FALSE, est_changes = 0L)`
   - Otherwise, for each CAS col check if any values would change under `ComptoxR::as_cas()`:
     - Values matching `^\\d+$` (unformatted pure digits) -> would change
     - Values matching placeholder patterns (case-insensitive "no cas", "n/a", "proprietary", "-", "none") -> would change to NA
   - `should_run <- est_changes > 0L`
   - Return result

4. **`precheck_name_cleaning(df, name_cols)`**
   - This is a combined pre-check for the entire name chain (Steps 6-pre through 6d3).
   - Check if `length(name_cols) == 0` -> `list(should_run = FALSE, est_changes = 0L)`
   - Check if all values in name_cols are NA or empty -> `list(should_run = FALSE, est_changes = 0L)`
   - Otherwise `should_run = TRUE`, `est_changes` = count of non-empty non-NA name values (all are candidates)
   - This is intentionally broad because individual name steps have complex interdependencies

5. **`precheck_isotope_shortcodes(df, name_cols, isotope_lookup)`**
   - If `is.null(isotope_lookup)` or `nrow(isotope_lookup) == 0`, return `list(should_run = FALSE, est_changes = 0L)`
   - Build regex from isotope_lookup shortcodes: `paste0("\\b(", paste(isotope_lookup$shortcode, collapse = "|"), ")\\b")`
   - For each name_col, count `stringr::str_detect()` matches
   - `should_run <- est_changes > 0L`

6. **`precheck_multi_analyte(df, name_cols)`**
   - Check for multi-analyte patterns: strings containing " and ", " & ", " / ", " + " between tokens
   - Use `stringr::str_detect(df[[col]], "\\s+(and|&|/)\\s+")` across name_cols
   - `should_run <- est_changes > 0L`

7. **`precheck_chiral_restore(df, name_cols)`**
   - Check if any values contain the chiral placeholder pattern `###CHIRAL_`
   - `should_run <- any(stringr::str_detect(df[[col]], "###CHIRAL_"), na.rm = TRUE)` across name_cols
   - `est_changes`: count of cells containing the placeholder

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pre-check -> step execution | Pre-check returning FALSE when step would have changed data causes silent data loss |
| est_changes accuracy | Over-counting is safe (cosmetic), under-counting is safe (step still runs), but should_run=FALSE when changes exist is catastrophic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-05 | Tampering | precheck returns FALSE incorrectly | mitigate | SKIP-03 false-negative companion tests for every precheck; test vectors that are edge cases for each detection primitive |
| T-37-06 | Denial of Service | precheck itself is expensive | accept | Pre-checks use cheap vectorized primitives (stringi::stri_enc_isascii, str_detect); cost is O(n) single-pass, negligible vs step cost |
| T-37-07 | Information Disclosure | est_changes leaks row counts | accept | est_changes is internal-only (not exposed to users); used for future RECO-01 modal in Phase 42 |
</threat_model>
