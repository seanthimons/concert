**1. `remap_audit_to_parent(audit_slice, parent_map)`**

Parameters:
- `audit_slice`: A 6-column audit tibble from the deduped unique-string slice (row_id values are 1..n_unique)
- `parent_map`: A named list where names are character positions in the unique slice (as strings "1", "2", ...) and values are integer vectors of ALL parent row indices that mapped to that unique value

Logic:
- If `nrow(audit_slice) == 0`, return `audit_slice` unchanged (already correctly typed empty tibble)
- For each row in `audit_slice`, look up `parent_map[[as.character(audit_slice$row_id[i])]]` to get the vector of parent row IDs
- Use pre-allocated vectors (same pattern as `build_audit_trail`): `all_row_ids <- integer()`, `all_fields <- character()`, etc.
- For each audit row, `rep()` all fields by `length(parent_indices)` and `c()` append to vectors
- Build single `tibble::tibble()` at end with the 6 columns
- Add assertion: `stopifnot(is.integer(result$row_id))` to enforce type safety
- Add `@export` roxygen tag

**2. `dedup_step(step_fn, df, ..., dedup_cols, uniqueness_threshold = 0.5)`**

Parameters:
- `step_fn`: The step function to call (e.g., `normalize_cas_fields`)
- `df`: The full parent dataframe
- `...`: Additional arguments passed to `step_fn` (e.g., `tag_map`, `name_cols`)
- `dedup_cols`: Character vector of column names to dedup on (e.g., `c("chemical_name")`)
- `uniqueness_threshold`: Numeric, default 0.5. If `n_distinct / n_total > threshold`, bypass dedup (per D-03)

Logic:
1. Extract the target columns from `df` as a single composite key: `paste0()` the dedup_cols values per row to create a key vector. Handle NAs by converting to `"__NA__"` sentinel before pasting.
2. Compute `n_distinct <- length(unique(key_vec))` and `n_total <- length(key_vec)`
3. **Uniqueness bypass** (D-03): If `n_distinct / n_total > uniqueness_threshold`, call `step_fn(df, ...)` directly and return its result unchanged.
4. Otherwise, build a dedup map:
   - `unique_keys <- unique(key_vec)`
   - `first_occurrence <- match(unique_keys, key_vec)` -- integer indices of first occurrence of each unique key
   - `df_unique <- df[first_occurrence, , drop = FALSE]` -- the deduped slice
   - `parent_map <- lapply(seq_along(unique_keys), function(i) which(key_vec == unique_keys[i]))` -- named by position in unique slice
   - `names(parent_map) <- as.character(seq_along(unique_keys))`
5. Call `result <- step_fn(df_unique, ...)`
6. Remap cleaned_data: for each row in `df`, find its unique-key index and copy the cleaned values from `result$cleaned_data` at that index. Use vectorized approach: `key_to_unique_idx <- match(key_vec, unique_keys)`, then `df_remapped <- result$cleaned_data[key_to_unique_idx, , drop = FALSE]`. Reset rownames.
7. Remap audit trail: call `remap_audit_to_parent(result$audit_trail, parent_map)`
8. Validate PERF-02 assertion: `if (nrow(remapped_audit) > 0) stopifnot(max(remapped_audit$row_id) <= nrow(df))`
9. Return `list(cleaned_data = df_remapped, audit_trail = remapped_audit)` -- same shape as direct step call.
10. Preserve any `new_tags` element if present in the step result: check `if (!is.null(result$new_tags)) return_list$new_tags <- result$new_tags`

Add roxygen documentation with `@export` tags for both functions.

```r
# Helper: simple uppercase step for testing dedup_step
uppercase_step <- function(df, cols) {
  df_result <- df
  audit_rows <- integer()
  audit_fields <- character()
  audit_originals <- character()
  audit_news <- character()
  for (col_name in cols) {
    original <- df[[col_name]]
    cleaned <- toupper(original)
    changed <- which(!is.na(original) & original != cleaned)
    if (length(changed) > 0) {
      audit_rows <- c(audit_rows, as.integer(changed))
      audit_fields <- c(audit_fields, rep(col_name, length(changed)))
      audit_originals <- c(audit_originals, original[changed])
      audit_news <- c(audit_news, cleaned[changed])
      df_result[[col_name]] <- cleaned
    }
  }
  list(
    cleaned_data = df_result,
    audit_trail = tibble::tibble(
      row_id = audit_rows, field = audit_fields,
      step = rep("uppercase", length(audit_rows)),
      original_value = audit_originals, new_value = audit_news,
      reason = rep("uppercased", length(audit_rows))
    )
  )
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dedup_step -> step_fn | dedup_step trusts that step_fn returns the correct contract; corrupted audit row_ids from step_fn could propagate |
| remap -> parent_map | parent_map keys must correspond exactly to audit_slice row_ids; mismatch causes silent data loss |
| dedup key construction | NA values and empty strings in dedup columns could cause incorrect grouping |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-01 | Tampering | remap_audit_to_parent | mitigate | stopifnot(max(result$row_id) <= nrow(parent)) assertion in dedup_step after remap; test in test-dedup-infrastructure.R |
| T-37-02 | Information Disclosure | dedup key with NAs | mitigate | Convert NA to "__NA__" sentinel before paste0 key construction; test NA grouping explicitly |
| T-37-03 | Denial of Service | dedup on high-cardinality columns | mitigate | uniqueness_threshold bypass (D-03) skips dedup when n_distinct/n_total > 0.5, avoiding overhead on non-beneficial cases |
| T-37-04 | Tampering | audit row_id out of range | mitigate | PERF-02 stopifnot assertion; companion test that verifies the assertion fires on invalid input |
</threat_model>
