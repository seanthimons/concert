<objective>
Apply unit-key dedup to `harmonize_units()` so that conversion factors are computed once per distinct unit combination, then broadcast-multiplied to all matching rows (PERF-03). This is the harmonization-side performance optimization parallel to the cleaning pipeline dedup in Plan 03.

Purpose: At 100K rows with typical regulatory data (50-100 unique unit strings), this eliminates redundant hash lookups and ppx/molarity classification for duplicate unit keys. The numeric value is excluded from the dedup key (D-07) since multiplication is O(1) vectorized.

Output: Optimized `harmonize_units()` in `R/unit_harmonizer.R` with dedup tests.
</objective>

dedup_keys[standard_mask] <- normalized[standard_mask]
dedup_keys[ppx_mask] <- paste0(normalized[ppx_mask], "||", media_vec[ppx_mask])
dedup_keys[molarity_mask] <- paste0(normalized[molarity_mask], "||", mw_vec[molarity_mask])

unique_keys <- unique(dedup_keys)
n_unique <- length(unique_keys)

# Only apply dedup optimization if worthwhile (more than 2x duplication)
if (n_unique < n / 2) {
  # Map each unique key to its first occurrence index
  first_idx <- match(unique_keys, dedup_keys)

  # Compute conversion results for unique keys only
  # We run the existing logic on a "unique" subset, then broadcast
  unique_normalized <- normalized[first_idx]
  unique_media <- media_vec[first_idx]

  unique_ppx_mask <- ppx_mask[first_idx]

  # Initialize per-unique results
  u_harmonized_unit <- orig_unit[first_idx]
  u_conversion_factor <- rep(1.0, n_unique)
  u_unit_flag <- rep("", n_unique)

# Phase 37: Unit-key dedup tests (PERF-03)

test_that("harmonize_units with high duplication produces identical results to direct", {
  unit_map <- load_unit_map()
  # 100 rows, only 5 unique units -> dedup should fire
  values <- rep(c(1.0, 2.5, 0.1, 100, 50), 20)
  units <- rep(c("mg/L", "ug/L", "ng/L", "mg/kg", "g/L"), 20)

  result <- harmonize_units(values, units, unit_map)

  expect_equal(nrow(result), 100)
  # All mg/L rows should have conversion_factor = 1
  expect_true(all(result$conversion_factor[units == "mg/L"] == 1))
  # All ug/L rows should have same conversion factor
  ug_factors <- unique(result$conversion_factor[units == "ug/L"])
  expect_length(ug_factors, 1)
  # Output schema correct
  expect_named(result, c("orig_row_id", "orig_unit", "harmonized_value",
                          "harmonized_unit", "conversion_factor", "unit_flag"))
})

test_that("harmonize_units dedup with ppx and media produces correct per-key results", {
  unit_map <- load_unit_map()
  # Same ppb unit but different media -> different keys
  values <- c(10, 10, 10, 10)
  units <- c("ppb", "ppb", "ppb", "ppb")
  media <- c("aqueous", "aqueous", "solid", "solid")

  result <- harmonize_units(values, units, unit_map, media = media)

  # Aqueous ppb -> mg/L, solid ppb -> mg/kg
  expect_equal(result$harmonized_unit[1], result$harmonized_unit[2])  # same media -> same target
  expect_equal(result$harmonized_unit[3], result$harmonized_unit[4])  # same media -> same target
  # Different media -> potentially different targets
  # (both convert to same factor 0.001 but different target units)
})

test_that("harmonize_units dedup with all unique units bypasses dedup", {
  unit_map <- load_unit_map()
  # Each unit is unique -> no dedup benefit -> bypass

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dedup key construction | Incorrect key grouping (e.g., missing media in ppx key) causes wrong conversion factors applied to wrong rows |
| broadcast via match() | If match() returns NA for any row, that row gets NA conversion factor and silent data corruption |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-11 | Tampering | Unit-key groups wrong rows together | mitigate | ppx keys include media, molarity keys include mw; test with mixed media/unit inputs verifies distinct results per group |
| T-37-12 | Tampering | match() returns NA for unknown key | mitigate | unique_keys is derived from dedup_keys, so match is guaranteed to find every key; but add defensive check: `stopifnot(!any(is.na(key_to_unique)))` |
| T-37-13 | Denial of Service | paste0 on large vectors | accept | paste0 is O(n) vectorized; for 100K rows this is < 1ms; the dedup saves far more than it costs |
</threat_model>
