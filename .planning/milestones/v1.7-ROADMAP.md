# Milestone v1.7: UI Polish & Isotope Cleaning

**Status:** ✅ SHIPPED 2026-04-13
**Phases:** 22-23
**Total Plans:** 3

## Overview

v1.7 delivered two focused improvements to ChemReg: UI polish to fix truncated column headers in Review Results and silence two console warnings, plus three new pre-curation cleaning functions (chiral protection, isotope shortcode expansion, and multi-analyte flagging) added to the 15-step cleaning pipeline. The chiral placeholder restore path was added as a post-audit tech debt fix before archiving.

### Phase 23: Isotope Cleaning

**Goal**: Users' chemical name columns are cleaned of isotope shortcodes before the bare formula detection step runs, with chiral designation protection and multi-analyte flagging also added to the pipeline
**Depends on**: Phase 22
**Requirements**: ISOT-01, ISOT-02, ISOT-03, CHIR-01, MANA-01
**Plans**: 2 plans

Plans:
- [x] 23-01-PLAN.md — Implement three new cleaning functions (chiral protection, isotope expansion, multi-analyte flagging) with unit tests
- [x] 23-02-PLAN.md — Wire functions into pipeline orchestrators, add integration tests, run smoke test

**Details:**

Success Criteria:
1. Cleaning pipeline audit trail shows isotope expansion step before bare formula detection
2. Isotope shortcodes (e.g., `u234`, `pb210`) expanded to full element names (e.g., `Uranium-234`, `Lead-210`)
3. Carbon backbone patterns and deuterium d-prefix patterns not incorrectly expanded
4. Only shortcodes under 5 characters from the ComptoxR known isotope list matched
5. Chiral designations protected from enclosure stripping and flagged as WARNING
6. Multi-analyte expressions (naked `+` or `and`) flagged as WARNING without auto-splitting

## Milestone Summary

**Key Decisions:**

- `protect_chiral_designations` uses content-encoded placeholder tokens (`###CHIRAL_PLUS###`, `###CHIRAL_R###`) rather than sequential numbers — enables stateless restore without row-index tracking (tech debt fix post-audit)
- `restore_chiral_designations()` wired as Step 9 in both `run_cleaning_pipeline()` and `mod_clean_data.R`, after `flag_multi_analyte` and before `detect_bare_formulas`
- `elementId` removal from reactable is safe: `session$ns("curation_table")` produces exactly the same string Shiny auto-assigns as HTML ID; Reactable.setFilter calls unaffected
- `unname(unlist(queue))` pattern applied before passing unlist() result through Shiny output bindings to fix jsonlite 2.0.0 named vector deprecation warning
- ComptoxR::pt$isotope direct access for isotope lookup; greedy element matching (sort by symbol length desc) for Pb before P
- ELEMENT_ALT_NAMES table added for American vs IUPAC name variants (cesium/Caesium)
- Source-code assertion tests (readLines + grep) valid for testing Shiny parameter choices that can't be tested without a full session

**Issues Resolved:**

- Column header truncation with ellipsis in Review Results DT table
- renderWidget explicit widget ID console warning
- jsonlite 2.0.0 named vector deprecation warning in curation workflow
- Chiral compounds failing ComptoxR curation due to `###CHIRAL_n###` placeholder leaking into API search

**Issues Deferred:**

- Human verification of wrap/warning behavior at runtime (requires live Shiny + ComptoxR session) — Low severity
- `test_cleaning_reference.R` 1 pre-existing failure (expects 3 keys, gets 4 including strip_terms) — pre-existing, low severity

**Technical Debt Incurred:**

- Nyquist VALIDATION.md files not created for Phases 22 or 23 — coverage gap for automated validation
