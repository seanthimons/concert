## Interaction Contract

Phase 22 is a pure fix pass — no new interactive elements. The contracts below govern the modified behaviors.

### UIPOL-01: Column Header Wrapping

| Property | Before | After | Decision Source |
|----------|--------|-------|-----------------|
| `wrap` parameter | `FALSE` | `TRUE` | D-01, CONTEXT.md |
| Header overflow | truncated with ellipsis | wraps to multiple lines | UIPOL-01 requirement |
| Optional CSS | none | `headerStyle = list(fontSize = "0.85em", whiteSpace = "normal")` on `defaultColDef` if headers exceed 3 lines | RESEARCH.md Pitfall 3 |
| Scope | `R/modules/mod_review_results.R` line 754 only | — | D-01 |

Visual rule: Header row height must not exceed 3 wrapped lines for any column. If it does, apply `defaultColDef(headerStyle = list(fontSize = "0.85em"))` as the fix. Test with actual curation data before committing.

### UIPOL-02: Remove elementId

| Property | Before | After | Decision Source |
|----------|--------|-------|-----------------|
| `elementId` in reactable call | `elementId = table_id` | removed | D-02, CONTEXT.md |
| JS filter behavior | `Reactable.setFilter(table_id, ...)` | unchanged — `table_id` still correct | RESEARCH.md |
| Console warning | `renderWidget` warning fires on render | silenced | UIPOL-02 requirement |

No visual change. No interaction change. Pure console noise fix.

### UIPOL-03: jsonlite Named Vector Warning

| Property | Behavior | Decision Source |
|----------|----------|-----------------|
| Named vectors in Shiny outputs | Convert to `unname()` or `as.list()` before passing | D-03, CONTEXT.md |
| Tracing approach | `options(warn = 2)` in live Shiny session | RESEARCH.md + CONTEXT.md |
| Confirmed site | `mod_review_results.R:1129` — `unname(unlist(queue))` | RESEARCH.md |
| Scope | Any named vector flowing into a `renderUI`/`reactive` output | D-03 |

No visual change. Console warning silenced.
