    **Part B: Implement expand_isotope_shortcodes()**

    Function `expand_isotope_shortcodes(df, name_cols)`:
    - Build lookup table from `ComptoxR::pt$isotope`:
      ```r
      isotopes <- ComptoxR::pt$isotope
      lookup <- tibble::tibble(
        shortcode = tolower(paste0(isotopes$element, isotopes$Z)),
        element_name = isotopes$Name,  # Full element name e.g. "Uranium"
        mass_number = as.character(isotopes$Z),
        element_symbol = isotopes$element,
        canonical = paste0(isotopes$Name, "-", isotopes$Z)  # "Uranium-234"
      )
      ```
      Wait — Z is atomic number, not mass number. Re-read the data: `ComptoxR::pt$isotope` has columns Z, element, Name, smiles, DTXSID. The isotope name column (Name) contains the mass info. Let me clarify: the shortcode column in the data likely encodes mass number, not Z. The executor MUST first inspect `head(ComptoxR::pt$isotope, 20)` to confirm column meanings. The shortcode is `paste0(Z, element)` per D-01 where Z is the mass number column. Build accordingly.

    - Sort lookup by `nchar(element_symbol)` descending for greedy matching (per D-04: match Pb before P)
    - Filter to element symbols < 5 chars (per D-07, though all standard symbols are 1-2 chars)

    Three-pass approach on each cell value:
    1. **Naked shortcode expansion**: For standalone tokens (word boundaries), match `\\b{element_symbol}{mass_number}\\b` case-insensitively. Replace with canonical `Name-Mass`. But first check it's not part of a molecular formula — skip if the token is followed by more element-like characters (uppercase letter) without whitespace. Specifically:
       - Match pattern: `(?i)\\b(element)(mass)\\b` where the match is NOT preceded by `[A-Z]` and NOT followed by `[A-Z]` (to exclude formula contexts like C12H22O11)
       - For carbon specifically (ISOT-03): skip if pattern is `C\\d+` and followed by `[A-Z]` (indicating formula like C12H22O11)
    2. **Spelled-out normalization** (per D-03): Match `{element_name}\\s+{mass_number}` and `{element_name}-{mass_number}` case-insensitively. Replace with canonical `Name-Mass` (properly capitalized).
    3. **Special cases**:
       - `unat` → do NOT expand, set `cleaning_flag` to `"WARNING: unresolvable isotope (unat)"` (per D-06)
       - `d-glucose` pattern: skip if `d-` is followed by a word (glucose, mannose, etc.) — this is a stereochemistry prefix (per ISOT-03)
       - `14C-glucose` pattern: if shortcode is prefix of a hyphenated compound name, do NOT expand (per D-02)

    - Record audit trail: step = "expand_isotope_shortcodes", original/new values, reason = "Isotope shortcode expanded in {col}" or "Isotope form normalized in {col}"
    - Return `list(cleaned_data = df_result, audit_trail = audit_trail)`
