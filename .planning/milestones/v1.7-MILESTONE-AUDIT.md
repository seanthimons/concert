### Missing Connection (Tech Debt)

**Chiral placeholder restore path — CHIR-01 (integration gap)**

`protect_chiral_designations()` replaces `(R)`, `(S)`, `(+)`, `(-)`, etc. with `###CHIRAL_n###` placeholders to prevent `strip_terminal_enclosures()` from deleting them. This is wired correctly for that purpose.

However, no `restore_chiral_designations()` function exists anywhere in the codebase. The placeholder strings persist through the entire downstream pipeline:
- `run_curation_pipeline()` receives `###CHIRAL_1###-catechin` as the name string
- ComptoxR exact search finds no match → row marked `unresolvable`
- Sheet 2 (Curated Data) displays placeholder strings verbatim

**Impact:** Chiral compounds will fail curation with "unresolvable" status. False negatives — these may match in ComptoxR if the clean name were submitted.

**Severity:** Medium — chiral designation protection is a relatively rare case in most chemical inventories, but it is a systematic failure for any inventory containing `(R)`/`(S)` nomenclature (stereospecific APIs, pharmaceutical ingredients).

**Precedent:** The `@@@`/`%%%` IUPAC comma protection pattern (Phase 12/19) IS restored — `restore_iupac_commas()` is called in `split_synonyms()`. The chiral protection omits this step.

**Suggested fix:** Add `restore_chiral_designations()` call in both `run_cleaning_pipeline()` (after `split_synonyms` or at end of pipeline before return) and `mod_clean_data.R` (before `detect_bare_formulas`). Reference the `@@@` restore pattern already implemented.
