### Critical Gaps

#### INT-01: ^tests$ in .Rbuildignore blocks R CMD check from running tests

**Severity:** CRITICAL
**Affected Requirements:** TST-03, TST-04

**What happened:** Phase 25 added `^tests$` to `.Rbuildignore` as a workaround for legacy `tests/test_*.R` files that crashed in R CMD check due to `source(here::here(...))` calls. Phase 28 migrated all tests to proper `tests/testthat/` structure and removed the problematic `source()` calls, but did not remove the workaround from `.Rbuildignore`.

**Impact:**
- `devtools::test()` works correctly (runs against live source tree) — Phase 28 passed with 953 tests
- `devtools::check()` and `R CMD check` run ZERO tests because the entire `tests/` directory is excluded from the source tarball
- A CRAN submission or CI/CD check would appear to have no tests

**Fix:** Remove `^tests$` from `.Rbuildignore`

---

### Moderate Gaps

#### INT-02: R/archive/prototype_pipeline.R has bare library() calls

**Severity:** MODERATE
**Affected Requirements:** SRC-04

**What happened:** `R/archive/prototype_pipeline.R` contains bare `library()` calls (lines 6-8: tibble, dplyr, purrr) and is included in the `R/` directory, which means it's part of the package build. Phase 25's goal was to eliminate all bare `library()` calls from `R/*.R` files, but this archive file was missed.

**Impact:**
- May produce R CMD check warnings about `library()` calls in package code
- Defines 6 functions that overlap with `R/curation.R` internal functions

**Fix:** Add `^R/archive$` to `.Rbuildignore`

---

### Minor Gaps

#### INT-03: Redundant library(withr) in 2 test files

**Severity:** MINOR
**Affected Requirements:** TST-01

**What happened:** `test-cleaning-reference.R` and `test-reference-provenance.R` both have `library(withr)` at line 4, but all withr function calls already use the `withr::` prefix. This is inconsistent with the Phase 28 cleanup standard of removing library() calls.

**Impact:** Non-breaking, but stylistically inconsistent

**Fix:** Remove `library(withr)` from both files
